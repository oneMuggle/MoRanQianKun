## 1. 快速开始

### 1.1 环境要求

- Node.js 20+
- npm 10+

### 1.2 安装与启动

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

默认地址：http://localhost:3000

### 1.3 界面布局

游戏界面分为三个主要区域：

| 区域 | 位置 | 功能 |
|------|------|------|
| 顶部栏 | 页面顶部 | 显示角色属性（境界、生命、时间等） |
| 左侧栏 | 页面左侧 | 聊天记录、历史剧情 |
| 右侧栏 | 页面右侧 | 功能菜单按钮（战斗、装备、背包等） |

### 1.4 基本操作

- **发送消息**：在底部输入框输入文字，点击发送或按 Enter
- **选择动作**：点击AI提供的动作选项按钮
- **滚屏查看**：使用鼠标滚轮或触屏滑动查看历史记录
- **打开功能面板**：点击右侧栏对应的功能按钮

## 2. 核心玩法

### 2.1 开局创建

点击界面上的「新游戏」或「开局」按钮开始新游戏。

**创建流程**：
1. 选择是否创建新世界或使用现有世界
2. 输入主角名称
3. 选择初始设定（出身、门派、境界等）
4. 确认后AI生成开局剧情

### 2.2 剧情推进

游戏采用回合式交互：

1. **你发送动作**：输入想要做的动作（如「走出房间」「与NPC对话」「查看包裹」等）
2. **AI生成回应**：根据你的动作生成对应的剧情发展
3. **选择选项**：AI可能提供多个动作选项供你选择

### 2.3 世界演变

游戏世界会根据时间推移和你的行为发生演变：

- **时间推进**：游戏内时间会自动流逝
- **世界变化**：NPC状态、门派势力、江湖动态会逐渐变化
- **演变触发**：点击「世界演变」按钮可手动触发世界更新

### 2.4 战斗系统

点击右侧栏「战斗」按钮进入战斗模式。

**战斗流程**：
1. 选择攻击、防守、使用功法、使用物品等动作
2. 根据属性判定战斗结果
3. 胜利可获得奖励，失败可能受伤

## 3. 功能面板指南

以下功能通过**右侧栏按钮**打开：

### 3.1 角色面板

**入口**：点击顶部栏角色名称查看详情

**功能**：
- 查看角色属性（生命、真气、境界、根骨等）
- 查看已装备的功法
- 查看角色状态和buff

### 3.2 物品系统

| 功能 | 入口 | 说明 |
|------|------|------|
| 背包 | 「背包」按钮 | 存储和查看物品 |
| 装备 | 「装备」按钮 | 穿戴武器、防具、饰品 |

**使用物品**：打开背包面板后，点击物品选中，再点击「使用」或「穿戴」

### 3.3 社交系统

**入口**：「社交」按钮

**功能**：
- 查看好友列表
- 管理NPC关系
- 与NPC进行互动（送礼、对话、切磋等）
- 管理 изображ（角色图像）

### 3.4 门派系统

**入口**：「门派」按钮

**功能**：
- 创建新门派
- 升级门派
- 招收弟子
- 管理门派事务

### 3.5 任务系统

**入口**：「任务」按钮

**功能**：
- 查看主线任务
- 查看支线任务
- 任务进度追踪

### 3.6 世界书

**入口**：「世界书」按钮（部分版本显示为「设定」）

**功能**：
- 编辑世界设定
- 添加自定义规则
- 注入NPC信息
- 修改场景描述

> **提示**：世界书是控制AI生成内容的重要工具，合理使用可以增强游戏体验。

### 3.7 地图系统

**入口**：「地图」按钮

**功能**：
- 查看当前所在位置
- 切换到其他已解锁地点
- 查看世界地理分布

### 3.8 记忆系统

**入口**：「记忆」按钮

**功能**：
- 查看游戏记忆摘要
- 手动触发记忆召回
- 回顾重要剧情节点

### 3.9 功法系统

**入口**：「功法」按钮

**功能**：
- 学习新功法
- 查看已学会的功法
- 修炼提升功法等级

### 3.10 队伍系统

**入口**：「队伍」按钮

**功能**：
- 创建队伍
- 招收队员
- 分配队伍任务

### 3.11 音乐系统

系统自动播放背景音乐。

如果想要切换音乐，可查看设置中的音乐选项。

### 3.12 约定系统

**入口**：「约定」按钮

**功能**：
- 自定义游戏规则
- 设置生成偏好
- 添加特殊约束

### 3.13 小说分解

**入口**：「小说分解」按钮（部分版本显示为「工作台」）

**功能**：
- 导入外部小说文本
- 分解剧情结构
- 将小说内容融入游戏

### 3.14 存档与读档

| 功能 | 入口 |
|------|------|
| 存档 | 「保存进度」按钮 |
| 读档 | 「读取进度」按钮 |

## 4. 图像系统

### 4.1 角色生成

可以为以下角色生成图像：

| 角色类型 | 说明 |
|----------|------|
| 主角 | 你的角色形象 |
| NPC | 江湖中的其他角色 |
| 香闺秘档 | 特殊亲密互动的角色 |

**生成流程**：
1. 打开对应的面板（社交、角色等）
2. 找到「生成图像」或「生图」按钮
3. 等待AI生成图像

### 4.2 场景生图

为当前所在场景生成背景图像。

### 4.3 图片管理

**入口**：「图册」或「 ImageManager 」按钮

**功能**：
- 查看所有已生成的图像
- 设定角色锚点（指定图像作为某角色的形象）
- 导出/删除图像

## 5. 存档与同步

### 5.1 本地存档

游戏数据默认存储在浏览器 IndexedDB 中。

**存档内容**：
- 角色状态
- 世界状态
- 物品背包
- 设置配置
- 生成的图像资源

### 5.2 云同步（可选）

需要配置 GitHub OAuth 才能使用云同步功能。

**配置步骤**：
1. 在设置中配置 GitHub Client ID
2. 首次同步时需要 GitHub 账号授权
3. 选择或创建私有仓库
4. 数据会自动同步到 GitHub Release

**可同步内容**：
- 存档数据
- 游戏设置
- 图像资源

**不同步内容**：
- 提示词池配置
- API 密钥

## 6. 设置与配置

### 6.1 入口

点击右侧栏「江湖设置」或「系统设置」按钮打开设置面板。

### 6.2 模型配置

在设置中可以配置：

- **文本模型**：选择用于剧情生成的AI模型
- **图片模型**：选择用于图像生成的AI模型（如 NovelAI）
- **API 地址**：配置模型服务的接口地址
- **API 密钥**：配置访问密钥

### 6.3 提示词调整

| 配置项 | 说明 |
|--------|------|
| 世界书 | 当前世界的设定和规则 |
| 酒馆预设 | 预设的角色和场景 |
| 提示词池 | 自定义提示词集合 |

### 6.4 显示设置

- **主题**：选择界面色调
- **字体大小**：调整显示字号

## 7. 常见问题

### Q1: 游戏无法启动？

检查 Node.js 版本（需要 20+）和 npm 版本（需要 10+）。

```bash
node --version
npm --version
```

### Q2: 界面显示异常？

1. 尝试清除浏览器缓存
2. 尝试使用其他浏览器（推荐 Chrome/Edge）
3. 检查控制台是否有错误信息

### Q3: AI 生成很慢或失败？

1. 检查 API 配置是否正确
2. 检查 API 密钥是否有效
3. 尝试切换到其他模型

### Q4: 存档丢失？

游戏数据存储在浏览器 IndexedDB 中，清除浏览器数据会导致存档丢失。

**建议**：定期使用「保存进度」功能备份存档。

### Q5: 如何修改世界书？

1. 打开「世界书」面板
2. 点击「编辑」按钮
3. 输入新的设定内容
4. 保存

### Q6: 如何获得更好的生成效果？

1. **细化世界书**：提供更详细的世界设定
2. **使用约定**：设置特定的生成约束
3. **调整提示词池**：添加针对性的提示词

### Q7: 图像生成失败？

1. 确保图片模型配置正确
2. 检查提示词是否包含禁止内容
3. 尝试简化提示词

### Q8: 游戏支持移动端吗？

游戏支持移动端，但部分功能在移动端可能有所简化。建议使用桌面端获得最佳体验。
