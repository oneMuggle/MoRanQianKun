# 天赋与背景系统分析文档

## 概述

本文档分析「墨色江湖：无尽武林」项目中天赋（Talent）和出身背景（Background）的定义方式、存储结构以及在游戏中的生效机制。

---

## 1. 数据结构定义

### 1.1 类型定义 (`types.ts`)

天赋和背景使用简洁的「名称-描述-效果」三元结构：

```typescript
// types.ts (第 88-98 行)
export interface 天赋结构 {
    名称: string;
    描述: string;
    效果: string; // 具体数值或逻辑描述
}

export interface 背景结构 {
    名称: string;
    描述: string;
    效果: string;
}
```

### 1.2 角色数据集成 (`models/character.ts`)

天赋列表和出身背景是角色数据的核心字段：

```typescript
// models/character.ts (第 65-67 行)
// New: Talents and Background
天赋列表: 天赋结构[];
出身背景: 背景结构;
```

## 3. 生效机制

### 3.1 核心设计原则：提示词驱动

天赋和背景**不通过硬编码逻辑生效**，而是通过以下方式影响游戏：

#### 3.1.1 系统提示词注入 (`hooks/useGame/systemPromptBuilder.ts`)

在构建角色状态文本时，天赋信息会被注入到系统提示词中：

```typescript
// hooks/useGame/systemPromptBuilder.ts (第 391-395 行)
const 天赋列表 = 取数组(role?.天赋列表).map((item: any) => ({
    名称: 取文本(item?.名称),
    描述: 取文本(item?.描述),
    效果: 取文本(item?.效果)
}));
```

这些信息最终会出现在发送给 AI 的 `角色状态` 块中。

#### 3.1.2 开局剧情生成 (`hooks/useGame/openingStoryWorkflow.ts`)

在开局剧情生成时，天赋信息作为建档摘要的一部分提供给 AI：

```typescript
// hooks/useGame/openingStoryWorkflow.ts (第 209-241 行)
const 构建开局角色建档摘要 = (roleData: any) => {
    const 天赋列表 = Array.isArray(roleData?.天赋列表)
        ? roleData.天赋列表.map((item, index) =>
            `[${index}] 名称：${名称}｜描述：${描述}｜效果：${效果}`
        ).join('\n')
        : '';

    return [
        `天赋数量：${roleData.天赋列表?.length || 0}`,
        `天赋详情：\n${天赋列表}`,
        `出身背景名称：${背景名称}`,
        `出身背景描述：${背景描述}`,
        `出身背景效果：${背景效果}`
    ].join('\n');
};
```

### 3.2 提示词约束 (`prompts/runtime/opening.ts`)

为确保天赋不被 AI 擅自修改，提示词中包含严格约束：

```
- 若建档、预设或输入中已明确给出 `角色.天赋列表[i].名称/描述/效果`，
  初始化时严禁擅自改名、改效果、改描述、偷换语义或做等价改写
- 天赋列表的 名称/描述/效果 视为不可擅改的正式真值
```

### 3.3 世界观生成影响 (`prompts/runtime/worldGeneration.ts`)

天赋信息在世界生成阶段作为边界约束：

```typescript
// prompts/runtime/worldGeneration.ts (第 96-117 行)
const 天赋列表 = charData.天赋列表.map((item) => item?.名称).join('、');
// 输出: 「天赋：天生剑骨、百工底子」
```

### 7.2 性别过滤机制

在 `NewGameWizard.tsx` 中，天赋和背景选项会根据角色性别动态过滤：

```typescript
const 全部背景选项 = useMemo(() => {
    const combined = [...预设背景, ...自定义背景列表];
    return combined.filter(item =>
        !item.适用性别 || item.适用性别 === charGender
    );
}, [自定义背景列表, charGender]);

const 全部天赋选项 = useMemo(() => {
    const combined = [...预设天赋, ...自定义天赋列表];
    return combined.filter(item =>
        !item.适用性别 || item.适用性别 === charGender
    );
}, [自定义天赋列表, charGender]);
```

### 7.3 切换性别时自动重置

当用户切换性别时，如果当前选择的天赋/背景不再适用，会自动重置：

```typescript
const 选择性别 = (next: '男' | '女' | '自定义') => {
    // ...
    setSelectedBackground((prev) => {
        if (prev?.适用性别 && prev.适用性别 !== next) {
            return 预设背景.find(b => !b.适用性别 || b.适用性别 === next) || 预设背景[0];
        }
        return prev;
    });
    setSelectedTalents((prev) => {
        return prev.filter(t => !t.适用性别 || t.适用性别 === next);
    });
};
```

## 8. 总结

天赋和背景系统的设计特点：

1. **声明式设计**：通过「名称-描述-效果」描述性字段定义，而非硬编码数值
2. **提示词驱动**：效果通过系统提示词传递给 AI，由 AI 在生成剧情时灵活运用
3. **不可篡改性**：提示词约束确保 AI 不会擅自修改玩家选择的天赋
4. **长期生效**：「长期」修饰的效果意味着贯穿整个游戏流程的影响
5. **灵活解释**：效果描述使用「提升概率」「更容易」「更难」等模糊表述，给 AI 留有演绎空间
6. **性别敏感**：通过 `适用性别` 字段实现性别相关的过滤与显示控制
