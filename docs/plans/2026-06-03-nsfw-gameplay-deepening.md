## 二、设计理念

### 2.1 目标

将 NSFW 从"被动阅读"升级为"**主动游玩**"——让玩家在做选择、管理资源、探索发现、应对后果的过程中，获得更深层的沉浸感和代入感。

### 2.2 原则

1. **不破坏叙事流畅性**——所有玩法层不中断 AI 叙事，而是增强 AI 叙事
2. **渐进式曝光**——玩家不必主动使用任何玩法，但可以选择深入
3. **选择有代价**——每个 NSFW 互动消耗资源，产生后果，带来奖励
4. **NPC 有主动性**——NPC 不仅是被动对象，也有欲望、动机、自主行为

### 2.3 核心玩法循环（目标状态）

#### B3. 里程碑检查机制

在游戏循环的回合结算阶段调用 `milestoneTracker.ts`：

```typescript
// 每回合结算时
function 结算回合奖励() {
  // 收集当前统计
  const 当前统计 = 收集NSFW统计数据(); // 性癖解锁数/敏感点发现数/亲密次数等

  // 检查成就
  const 新成就 = 检查成就解锁(当前统计, 已解锁成就);

    发放成就奖励(成就); // 代币/场景/技巧
    推送成就通知(成就); // 游戏内通知
  }

  // 检查里程碑
  const 新里程碑 = 检查里程碑进度(当前统计);
  for (const 里程碑 of 新里程碑) {
    推送里程碑提示(里程碑); // 可选：成就面板中展示
  }
}
```

### 模块 C：后果应对系统

**现状**：`nsfwCore/consequenceResolution.ts` 定义了 7 种应对策略（辟谣/收买/转移注意力/利用后果/正面解决/法律手段/私下和解），有完整的成功率和资源消耗计算。`ConsequenceManager.tsx` 组件已写好（370行），0处导入。

**深化设计**：

#### C1. 后果触发流程

```
NSFW互动结束
    ↓
后果判定（基于风险预算/暴露等级/场景类型）
    ↓
触发后果？
    ├── 否 → 正常结算
    └── 是 → 弹出后果应对界面（ConsequenceManager）
                ↓
           玩家选择应对策略
                ↓
           成功率判定（受资源/属性/关系影响）
                ↓
           成功 → 后果消除/减轻 + 消耗资源
           失败 → 后果加重 + 额外惩罚
```

#### C2. 后果类型与应对

| 后果 | 严重程度 | 可用策略 | 成功率因素 |
|------|---------|---------|-----------|
| 暴露风险（被人看到） | 中~高 | 辟谣/收买/转移注意力 | 声誉值/代币/关系 |
| 流言传播 | 中 | 辟谣/私下和解/利用后果 | 声誉/法律记录 |
| 学业/事业下滑 | 高 | 正面解决/法律手段 | 属性/法律记录 |
| 多角冲突（NPC间冲突） | 中~高 | 私下和解/利用后果 | 和谐度/关系 |
| 法律风险 | 高 | 法律手段/收买/转移 | 法律记录/资源 |
| 道德谴责 | 中 | 正面解决/私下和解 | 道德评判/关系 |
| 心理创伤（NPC） | 高 | 温柔护理/时间治疗 | 事后护理质量 |

#### C3. UI 流程

```
┌──────────────────────────────────────┐
│  ⚠️ 后果出现                         │
├──────────────────────────────────────┤
│  暴露风险：在图书馆的亲密行为         │
│  被一名同学目击                       │
│                                      │
│  严重程度：中等                       │
│  潜在影响：流言传播 · 声誉-10         │
│                                      │
│  应对策略：                           │
│  ○ 辟谣 — 消耗20代币，成功率60%      │
│  ○ 收买 — 消耗30代币+5声誉，         │
│         成功率75%                    │
│  ○ 转移注意力 — 消耗10精力，         │
│         成功率50%                    │
│                                      │
│  [确认策略] [跳过（承受后果）]        │
└──────────────────────────────────────┘
```

**关键**：后果应对是"选择有代价"的核心体现。玩家可以选择跳过应对（被动承受），但后果会更严重。

#### D3. 心理防线机制

| 放纵 (0-19) | 95% | 85% | 主动配合 |

**降低防线的方法**：
- 提升亲密度等级（每级 -10）
- 温柔事后护理（-5）
- 多次 NSFW 互动自然衰减（每回合 -3）
- 特定场景/环境修饰（-5~15）

#### D4. 敏感点发现

与性癖探索类似，但关注 NPC 身体的敏感区域：

```

- 轻触试探（低风险，发现表层敏感点）

发现后效果：
- 该敏感点在后续 NSFW 互动中提供额外亲密度加成
- 影响 AI 叙事（更详细的敏感部位描写）
- 可能触发 NPC 特殊反应/情感变化
```

## 六、风险与约束

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 玩法系统过多干扰叙事流畅性 | 高 | 所有玩法均可关闭/跳过，保持"纯阅读"模式可用 |
| 资源系统过于复杂让玩家困惑 | 中 | 渐进式曝光，先展示4个核心资源，逐步解锁详情 |
| NPC 自主行为通知过于频繁 | 中 | 设置通知频率上限（每回合最多1个），可关闭 |
| 后果系统惩罚过重导致挫败 | 中 | 提供"跳过后果"选项（承受较轻惩罚） |
| 心理防线/性癖探索节奏过慢 | 低 | 提供洞察符道具加速 |
| 移动端 UI 过于拥挤 | 中 | 移动端使用底部抽屉+折叠面板 |
