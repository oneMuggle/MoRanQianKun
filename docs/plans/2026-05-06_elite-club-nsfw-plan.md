## 一、项目概述与愿景

### 1.1 项目定位

本系统定位为 **「江湖风高端会员制会所/SPA 一体化平台」**，面向高端消费群体，提供预约、管理、私密社交、定制服务等全链路体验。整体调性融合 **东方武侠美学** 与 **现代奢华极简**，强调私密性、尊贵感与仪式感。

系统分为 **C端（会员/客人）** 与 **B端（管理端）** 两大模块，支持 iOS/Android/H5/小程序多端访问。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **私密性** | 全流程隐私保护，会员信息加密，房间/包厢制预约 |
| **尊贵感** | 入会门槛分级（铜/银/金/钻石/至尊），差异化权益 |
| **仪式感** | 传统江湖文化元素嵌入服务流程（品茗、开背前静心仪式）|
| **智能化** | AI 推荐个性化疗程、房间自动调节（温/光/音）|
| **合规性** | 明确 NSFW 内容边界，18+ 年龄验证，内容审核机制 |

✗ 人口贩卖、强迫、胁迫
✗ 涉毒/涉赌内容

| 场景 | 描述 | 约束条件 |
|------|------|----------|
| **艺术按摩示意图** | 人体经络/穴位示意图，用于健康理疗科普 | 需 18+ 验证，图片须有艺术化处理（遮盖敏感部位）|

### 2.3 年龄验证流程

```
注册流程强制插入年龄验证节点：
[注册手机号] → [年龄输入+身份证 OCR 识别] → [公安部实名验证 API] 
→ [通过/拒绝] → [签署《服务协议与隐私条款》] → [完成注册]
```

18-21 岁用户：限制访问部分高价疗程（如"帝王秘境"等），界面提示"需至尊会员邀请"。

## 四、技术架构

### 4.1 技术栈建议

| 层级 | 技术选型 |
|------|----------|
| **前端** | React Native（iOS/Android）+ Taro（H5/小程序）|
| **后端** | Node.js（Koa）或 Java（Spring Boot）|
| **数据库** | PostgreSQL（主数据）+ Redis（缓存）+ 阿里OSS（文件存储）|
| **AI 服务** | 阿里云内容审核 API / 腾讯云鉴黄 API |
| **支付** | 微信支付/支付宝/银行卡，PCI-DSS 合规 |
| **短信** | 阿里云短信（验证码/通知）|
| **推送** | 极光推送 / Firebase |
| **监控** | Prometheus + Grafana |
| **日志** | ELK Stack（Elasticsearch+Logstash+Kibana）|

### 4.2 安全架构

```
[客户端] 
  ↕ HTTPS + TLS1.3
[API Gateway] （鉴权+限流+审计）
  ↕
[业务服务层] （微服务拆分）
  ↕
[数据层] （加密存储：AES-256，敏感字段加密存储）
```

- **传输加密**：全链路 TLS 1.3，证书双向认证
- **数据加密**：会员手机号、身份证、支付信息 AES-256 加密
- **日志脱敏**：操作日志自动打码，管理员不可查阅完整敏感数据
- **接口鉴权**：JWT + RefreshToken，15min 自动刷新
- **风控规则**：同一 IP 短时大量请求自动熔断

### 4.3 NSFW 审核技术方案

```javascript
// 内容审核流程伪代码
async function contentModeration(file, text) {
  // 1. 图片 AI 审核
  const imageResult = await aiService.checkImage(file); // 阿里云/腾讯云
  if (imageResult.label === 'normal') {
    // 2. 文字内容审核
    const textResult = await aiService.checkText(text);
    if (textResult.label === 'pass') {
      return { status: 'approved' };
    } else {
      return { status: 'manual_review', reason: textResult.label };
    }
  } else {
    // 3. 涉敏图片直接拒绝
    return { status: 'rejected', reason: imageResult.label };
  }
}
```

- **UGC 图片**：上传即触发 AI 审核，涉敏内容实时拦截
- **UGC 文本**：敏感词库 + AI 模型双重审核
- **人工复核队列**：AI 标记为"疑似"的内容进入人工审核池

## 六、UI/UX 设计规范

### 6.1 视觉风格

- **主色调**：玄黑 `#1A1A2E`、墨灰 `#16213E`、鎏金 `#D4AF37`
- **辅色**：宣纸白 `#F5F0E8`、朱砂红 `#9B2335`
- **字体**：思源宋体（标题）+ 思源黑体（正文）
- **图标**：线性图标 + 传统纹样（祥云/回纹/太极）

### 6.2 隐私设计要点

- 所有涉及身体部位的图片均使用 **艺术化插画/示意图**
- 评价展示时 **自动打码** 用户头像和昵称（管理员除外）
- 预约页面 **不显示** 其他人正在预约同一房间的信息
- 消息通知 **不暴露** 具体服务内容，仅显示"您预约的服务即将开始"

### 6.3 动效与反馈

- 疗程开始前：3秒静心引导动画（如水墨晕染）
- 支付成功：金色粒子庆祝动画
- 异常操作：红色/橙色警示，震动反馈

## 七、运营合规体系

### 7.1 内容运营红线

| 禁止 | 允许 |
|------|------|
| "特殊服务""大保健"等暗语 | "经络疏通""深层放松"等专业术语 |
| 技师真实照片（正面照）| 艺术化处理后的工作场景图 |
| 用户晒单暴露身体 | 晒单仅展示环境/产品/氛围图 |
| 展示现金/转账截图 | 展示会员积分/等级变化 |

### 7.2 举报与响应机制

```
用户举报 → 系统记录+截图 → 24h 内自动通知被举报方
→ 3 人评审小组投票 → 判定违规/误报
→ 违规：警告/扣分/封号 → 结果通知双方
→ 误报：隐藏举报记录，保护被举报方
```

### 7.3 监管对接

- 预留监管机构 API 查询接口（仅响应主管部门指令）
- 日志留存：操作日志保留 ≥ 180 天
- 营收数据定期上报（如有要求）
