# NPC 服饰系统详细改造计划

**日期**: 2026-05-11
**状态**: 已完成
**作者**: Claude Code (规划模式)

---

## 1. 背景与目标

### 1.1 现状分析

当前 NPC 的衣着描述在社交面板（SocialModal / MobileSocial）中呈现为**单一自由文本字段** `衣着风格`，显示效果为：

```
衣着: 一袭素白道袍，腰间系着青色丝绦
```

已有相关数据结构：

| 字段 | 位置 | 用途 |
|------|------|------|
| `衣着风格?: string` | `NPC结构` | 单一衣着描述（当前使用） |
| `当前服装状态?: 服装状态结构` | `NPC结构` | NSFW 服装穿脱状态（上衣/下装/内衣/内裤/袜饰） |
| `当前装备?: NPC装备栏` | `NPC结构` | 队友装备栏（服装/内衣/内裤/袜饰/鞋履/饰品） |

**关键问题**：
1. `衣着风格` 是扁平字符串，无法分部位管理
2. `NPC装备栏` 仅对 `是否队友=true` 的 NPC 有效，普通社交 NPC 无装备栏
3. `当前服装状态` 仅追踪穿脱状态（穿着/半敞/褪下/移除），不记录具体物品名称
4. 三者数据孤岛，无统一联动
5. 故事事件驱动服装变更缺乏系统性机制
6. **现代纪元 NSFW 适配缺失**：当前设计偏向古代武侠风格，无法覆盖现代 NSFW 场景中的器具/道具类物品（丝袜、跳蛋、肛塞、乳夹、皮质项圈等）。需要区分"覆盖身体的衣物"和"作用于身体的道具"

### 1.2 改造目标

1. **分部位结构化**: 将 NPC 衣着拆分为上衣、下着、鞋子、袜子、内衣、内裤、配饰等独立类别
2. **故事驱动变更**: 支持剧情事件自动更新 NPC 穿着（AI 生成、事件触发、玩家交互）
3. **UI 增强展示**: 社交面板按部位展示衣着详情
4. **向后兼容**: 现有存档数据平滑迁移，`衣着风格` 保留为向后兼容字段
5. **AI 提示词更新**: 指导 AI 在生成故事响应时正确维护和更新服装数据
6. **现代纪元 NSFW 适配**: 新增 `道具档案` 系统，与 `服饰档案` 并行，覆盖束缚器具、刺激器具、穿戴器具、消耗品等现代 NSFW 道具类别

## 2. 关键设计决策

### 2.1 结构化类型 vs 自由文本

**决策：混合方案 —— 每个部位采用 `{ 名称: string, 描述: string }` 结构**

理由：
- 纯结构化（枚举物品 ID）不适合本项目的开放性叙事风格
- 纯自由文本无法实现分部位管理和事件驱动更新
- 混合方案兼顾灵活性与可操作性

```typescript
interface 服装部位 {
    名称: string;     // 如 "素白道袍"、"蜀锦襦裙"
    描述: string;     // 如 "一袭素白道袍，袖口绣有青云纹"
}
```

### 2.2 故事驱动服装变更机制

**决策：通过 AI 生成的 tavern_commands 更新 + 事件触发辅助**

- 主要路径：AI 在生成故事响应时，根据剧情发展自动生成 `set 社交[i].服饰档案.上衣 = ...` 等命令
- 辅助路径：预设事件（如 "沐浴"、"战斗破损"、"换装"）可触发批量状态变更
- 变更粒度：AI 可选择更新单个部位或整体换装

### 2.3 移动端 vs 桌面端

**决策：统一数据模型，差异化 UI 展示**

- 桌面端（SocialModal）：多列网格展示各部位，支持展开/折叠
- 移动端（MobileSocial）：手风琴式折叠展示，默认仅显示摘要
- 两者共用同一 `读取服饰()` 辅助函数

### 2.4 向后兼容

**决策：保留 `衣着风格` 为派生字段 + 数据迁移逻辑**

- 新增 `服饰档案` 字段，不删除 `衣着风格`
- 读取时优先使用 `服饰档案`，回退到 `衣着风格`
- 旧存档加载时自动将 `衣着风格` 转为 `服饰档案.上衣` 的初始值
- `衣着风格` 可由 `服饰档案` 自动拼接生成（保持 prompt 兼容）

```typescript
/**
 * 将旧存档中的 衣着风格 迁移到 服饰档案
 */
export const 迁移服饰档案 = (npc: NPC结构): NPC结构 => {
    if (npc.服饰档案) return npc; // 已有新数据，跳过

    if (npc.衣着风格 && npc.衣着风格.trim()) {
        return {
            ...npc,
            服饰档案: {
                上衣: {
                    名称: npc.衣着风格.trim(),
                    描述: npc.衣着风格.trim()
                }
            }
        };
    }

    return npc;
};

};

/**
 * 获取特定部位的服装信息
 */
export const 获取服装部位 = (
    npc: NPC结构,
    部位: 服饰部位分类
): 服装部位 | undefined => {
    return npc.服饰档案?.[部位];
};

/**
 * 获取特定类别的道具信息
 */
export const 获取道具类别 = (
    npc: NPC结构,
    类别: 道具部位分类
): 道具条目 | undefined => {
    return npc.道具档案?.[类别];
};

/**
 * 获取 NPC 道具摘要文本（用于 UI 快速展示）
 */
export const 获取NPC道具摘要 = (npc: NPC结构): string => {
    const 档案 = npc.道具档案;
    if (!档案) return '';
    const parts: string[] = [];
    if (档案.束缚器具?.名称) parts.push(档案.束缚器具.名称);
    if (档案.刺激器具?.名称) parts.push(档案.刺激器具.名称);
    if (档案.穿戴器具?.名称) parts.push(档案.穿戴器具.名称);
    if (档案.遥控设备?.名称) parts.push(档案.遥控设备.名称);
    if (parts.length > 0) return parts.join('，');
    return '';
};
```

## 10. 风险与依赖

### 10.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI 生成的命令格式错误 | 服装/道具更新失败 | 在 `responseCommandProcessor` 中添加宽容解析 |
| 旧存档迁移遗漏 | 部分 NPC 丢失衣着 | 迁移函数对每个 NPC 独立执行，不批量中断 |
| UI 展示空间不足（移动端） | 信息拥挤 | 使用手风琴折叠，默认仅展示名称 |
| `服饰部位分类` / `道具部位分类` 枚举变更 | 类型不兼容 | 初始版本保持精简，后续迭代扩展 |
| Prompt 字符数膨胀 | 上下文窗口占用增加 | 使用紧凑 JSON 格式，仅在场 NPC 注入 |
| AI 生成不合时代背景的道具 | 沉浸感破坏 | 在 prompt 中明确区分古今道具词汇范围 |
| 道具状态与 `当前服装状态` 冲突 | 数据不一致 | 明确分工：`当前服装状态` 管穿脱，`道具档案` 管物品 |

### 10.2 依赖

- 依赖现有 `tavern_commands` 命令执行系统（已稳定运行）
- 依赖 `NPC结构` 现有字段（`衣着风格`、`当前服装状态`、`当前装备`）保持不变
- 不依赖新的后端服务或外部 API
- 需要与 NSFW 系统（`服装状态结构`）协调，确保不冲突
- 需要与 BDSM/campus NSFW 系统协调，确保道具档案与 SM 场景类型兼容

### 10.3 向后兼容性

- **存档兼容**: 旧存档加载后自动迁移，`衣着风格` 转为 `服饰档案.上衣`
- **Prompt 兼容**: AI 仍可读写 `衣着风格` 字段，作为 `服饰档案` 的补充
- **UI 兼容**: `读取衣着()` 函数回退到 `衣着风格`，确保无任何新数据时仍能展示
