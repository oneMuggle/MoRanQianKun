## 一、需求重述

1. 在 `contemporary_eastern`（东方现代）下新增 `contemporary_campus`（校园）子纪元
2. 强化里模式系统 — 从单条 rules 文本升级为结构化定义 + 三级强度控制

---

## 二、现状分析

### 当前里模式运行机制

| 维度 | 现状 |
|------|------|
| **数据结构** | 每个 SubEra 节点的 `liMode` 包含 `name` + `description` + `rules`（纯文本 markdown） |
| **注入时机** | `eraLiMode.ts` 解析 `rules` 字段，拼接到 system prompt（`systemPromptBuilder.ts:1439`） |
| **UI 控制** | 新游戏向导中一个"子纪元里模式"全局开关 |
| **NPC 绑定** | AI 自行决定哪些 NPC 有表里人格，无结构化绑定 |

### 已有 liMode 统计

| 时代 | liMode 数量 |
|------|------------|
| 古代 (ancient) | 10 |
| 当代 (contemporary) | 9 |
| 现代 (modern) | 6 |
| 近未来 (near-future) | 3 |
| 远未来 (far-future) | 3 |
| 后人类 (post-human) | 3 |
| 远古 (primordial) | 3 |

### 强化方向

| 维度 | 当前 | 强化后 |
|------|------|--------|
| **规则结构** | 单条长文本 rules | 拆分为结构化字段（核心原理/权力系统/表里人格/场景类型/欲望动机/禁忌/AI指令） |
| **NPC 绑定** | AI 自行决定 | 每个 characterArchetype 绑定表/里人格字段，运行时自动注入 |
| **分级控制** | 全局开/关 | 3 级强度：微暗 → 暧昧 → 露骨 |
| **运行时变量** | 固定文本 | 动态注入好感度/修为/权力等级等变量 |

## 三、实施步骤

### 步骤 1：扩展类型定义

**文件：** `models/eraTheme/types.ts`

- 新增 `EraLiModeEnhanced` 接口，定义结构化字段
- 将 `EraNode.liMode` 类型扩展为兼容两种格式
- 为 `EraCharacterArchetype` 添加可选的 `表人格` / `里人格` 字段

### 步骤 2：更新里模式注入逻辑

**文件：** `prompts/runtime/eraLiMode.ts`

- 优先读取结构化字段，组装为分层 prompt
- 无结构化字段则 fallback 到原有 `rules` 文本
- 根据强度级别（微暗/暧昧/露骨）动态裁剪注入内容
- 注入当前运行时变量

### 步骤 3：定义校园子纪元节点

**文件：** `models/eraTheme/epoch-contemporary.ts`

在 `contemporary_rural` 节点之后插入 `contemporary_campus`，包含：
- 基础配置（colors/typography/uiStyle/bgmTags/artStyle）
- UI 文案（全面校园化）
- 开局场景（6 个）
- 角色原型（6 个，含表里人格）
- 写作样例（2 个）
- 里模式（强化版，含三级强度）

### 开局场景（6 个）

1. `图书馆自习` — 期末季的图书馆座无虚席，邻座的目光交汇
2. `社团招新` — 百团大战，各显神通，命运的邂逅

### 角色原型（6 个）

1. `学霸` — 表：完美笔记、一丝不苟 / 里：中二病、渴望被认可
2. `社团达人` — 表：八面玲珑 / 里：内心孤独、渴望真诚关系
3. `隐形大佬` — 表：低调透明 / 里：掌控全局、深藏不露
4. `叛逆者` — 表：逃课常客 / 里：内心柔软、重情重义
5. `温柔学长` — 表：照顾新人、如沐春风 / 里：掌控欲强、引导型
6. `神秘转学生` — 表：背景成谜 / 里：主动出击、目标明确
