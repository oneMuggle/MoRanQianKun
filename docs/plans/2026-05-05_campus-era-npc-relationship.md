> 日期：2026-05-05

## 一、目标与背景

在现有校园纪元系统基础上，构建完整的 **NPC 关系系统**，实现玩家与 NPC 之间关系的结构化管理、递进式发展和多维度互动反馈。

### 核心设计原则

1. **关系结构化**：NPC 关系数据模型化，包含好感度、亲密度、关系类型、互动历史
2. **递进式发展**：关系从初识到深度绑定，有明确的发展阶段和解锁条件
3. **多维度互动**：支持多种互动方式（对话、礼物、任务、场景），各有不同效果
4. **校园场景融合**：关系发展与校园环境紧密结合（社团、教室、宿舍等）
5. **AI 驱动叙事**：关系变化融入主剧情，AI 根据关系状态调整互动内容

## 二、数据模型扩展

### 2.1 新增类型（`models/campusNSFW/relationship.ts`）

```typescript
export type 关系类型 = '陌生' | '相识' | '好感' | '亲密' | '恋人' | '挚友';
export type 关系状态 = '单恋' | '暧昧' | '已确认' | '分手' | '友谊';

export interface NPC关系数据 {
  npcId: string;
  关系类型: 关系类型;
  关系状态: 关系状态;
  好感度: number;           // 0-100
  亲密度: number;           // 0-100，影响解锁内容
  信任度: number;            // 0-100，影响 NPC 坦诚程度
  感情值: number;            // 0-100，情感深度
  最近互动时间?: string;
  互动次数: number;
  关键事件: 关系事件[];
  独家标记: boolean;        // 是否确立独占关系
  解锁场景: string[];       // 已解锁的私密场景
}

export interface 关系事件 {
  id: string;
  时间: string;
  类型: '初识' | '对话' | '礼物' | '任务' | '亲密' | '冲突' | '里程碑';
  标题: string;
  描述: string;
  好感度变化: number;
  亲密度变化: number;
  信任度变化: number;
  感情值变化: number;
}

export interface 关系阈值 {
  [关系类型: string]: {
    好感度: number;
    亲密度: number;
    信任度: number;
    感情值: number;
  };
}
```

## 六、关系阶段详解

### 6.1 阶段推进表

| 阶段 | 好感度 | 亲密度 | 信任度 | 解锁内容 |
|------|--------|--------|--------|----------|
| **陌生** | 0-19 | 0 | 0 | 基础对话 |
| **相识** | 20-39 | 10+ | 5+ | 课后闲聊、简单邀约 |
| **好感** | 40-59 | 25+ | 20+ | 送礼、私密场景解锁 |
| **亲密** | 60-79 | 50+ | 40+ | 深度对话、亲密互动 |
| **恋人** | 80-100 | 75+ | 60+ | 独占关系、特殊剧情 |

### 6.2 关系类型与状态组合

```
关系类型: 陌生 → 相识 → 好感 → 亲密 → 恋人/挚友
                ↓
关系状态: 单恋(双方未确认) / 暧昧(一方表白) / 已确认 / 友谊
```

## 十一、风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| AI 生成的关系内容不稳定 | 中 | 严格的关系变化数值约束 + 预设回退 |
| 关系数据与存档兼容性问题 | 低 | 新字段有默认值，向后兼容 |
| 多角关系复杂度爆炸 | 高 | 设置同时关系数量上限 |
| 关系系统与 BDSM 系统冲突 | 中 | 共享数据源，关系系统为基础层 |
