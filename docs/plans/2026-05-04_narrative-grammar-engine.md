# 叙事语法引擎 (Narrative Grammar Engine)

**日期**: 2026-05-04
**状态**: 已实现
**优先级**: 高

---

## 背景

游戏主剧情输出依赖 `prompts/core/format.ts` 中定义的叙事格式规则，包括：
- `<正文>` 标签内的三类合法行：`【旁白】`、`【角色名】`、`【判定】`
- `<judge>` 判定子结构
- `<变量规划>`、`<剧情规划>`、`<短期记忆>` 等辅助标签

现有实现将所有格式规则硬编码在提示词中，缺少独立的语法引擎层来：
1. 结构化解析和验证叙事输出
2. 规范化不规范的输入
3. 提取叙事元素供后续流程使用

---

## 实现方案

### 1. 类型定义 (`models/narrativeGrammar.ts`)

```typescript
// 叙事行类型
type 叙事行类型 = '旁白' | '角色台词' | '判定';

// 旁白行
interface 旁白行 {
  类型: '旁白';
  内容: string;
}

// 角色台词行
interface 角色台词行 {
  类型: '角色台词';
  角色名: string;
  内容: string;
}

// 判定行
interface 判定行 {
  类型: '判定';
  判定类型: '通用' | '对抗' | '洞察' | '先机' | '瞄准' | '接战' | '防御' | '伤害' | '态势' | '反击' | '反馈' | '消耗' | '衰退';
  行动名: string;
  触发对象: string;
  判定值: number;
  难度: number;
  基础: number;
  环境: number;
  状态: number;
  幸运?: number;
  装备?: string;
  结果: '成功' | '失败' | '大成功' | '大失败';
}

// 叙事块
interface 叙事块 {
  正文: (旁白行 | 角色台词行 | 判定行)[];
  变量规划?: string;
  剧情规划?: string;
  短期记忆?: string;
  judge?: string;
}
```

### 2. 核心模块 (`hooks/useGame/narrativeGrammar.ts`)

- `解析叙事块(文本)` → 叙事块
- `提取旁白行(文本)` → 旁白行[]
- `提取角色台词(文本)` → 角色台词行[]
- `提取判定行(文本)` → 判定行[]
- `验证叙事格式(叙事块)` → 验证结果
- `规范化叙事文本(原始文本)` → 规范化文本

### 3. 集成点

- 在 `responseProcessingPhase` 调用 `解析叙事块` 验证格式
- 将 `规范化叙事文本` 用于日志记录
- 提取的判定行数据可用于后续战斗/数值计算流程
