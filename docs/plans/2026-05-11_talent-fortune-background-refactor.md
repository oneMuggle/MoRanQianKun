# 天赋·气运·背景系统整理计划

## 背景与目标

项目中天赋（talents）、气运（qiyun/fortune）、背景（backgrounds）三大系统经过多轮迭代，存在以下问题：

1. **冗余项过多**：气运系统中 `zhen-qiyun.ts` 有 380+ 条目，大量"XX红利"模板化重复条目（如 `焊接红利`、`舞台红利`、`音乐红利` 等，仅职业前缀不同，效果完全一致）
2. **重复条目**：同一概念在多个文件中出现（如 `天煞孤星`、`热搜体质`、`节气感应` 等重复出现 2 次以上）
3. **武侠核心内容薄弱**：作为武侠题材游戏，天赋仅 5 条、背景仅 4 条，缺少内功、外功、兵器、门派、江湖恩怨等核心武侠概念
4. **规模失衡**：现代/未来题材各有 150+ 条目，武侠核心仅个位数，比例严重失调

**目标**：
- 去除/合并冗余项，精简气运系统（目标：zhen-qiyun 从 380+ 降至 ~100）
- 合并重复条目（统一入口，通过 era/tag 区分变体）
- 大幅增加武侠核心内容（天赋 5→30+，背景 4→30+）
- 补充缺失的武侠气运类别（内功心法、外功招式、兵器谱、门派渊源、江湖恩怨、奇遇秘境等）

## 技术方案

### 阶段一：冗余项分析与去重

#### 1.1 气运系统 `zhen-qiyun.ts` 去重

**"XX红利"模板化条目合并**：
- 约 60+ 条 `XX红利` 条目（焊接、舞台、音乐、舞蹈、宠物、烘焙、园艺、编程...），全部是 `+30%/+50% 相关技能检定`
- 合并为 5-8 条代表性条目：
  - `技艺红利`（通用手工艺：焊接/烘焙/园艺）
  - `表演红利`（舞台/音乐/舞蹈/戏剧）
  - `学术红利`（科研/编程/AI）
  - `社交红利`（网红/直播/自媒体）
  - `商业红利`（创业/投资/电商）
  - `运动红利`（体育/健身/极限运动）
  - `自然红利`（宠物/种植/狩猎）

**完全重复条目去重**：
- `热搜体质`：出现 2 次，保留 1 次
- `节气感应`/`土地公保佑`/`丰收之眼`/`五谷丰登`：现代 era 与 contemporary_rural sub-era 各出现 1 次，通过 `子纪元适配` 字段合并
- 其他描述高度相似、效果完全相同的条目合并

**目标**：zhen-qiyun.ts 从 380+ 条降至 ~100 条

#### 1.2 跨文件重复去重

- `天煞孤星` 概念在 `zhen-qiyun.ts` 和 `xianzhi.ts` 中各出现，保留一处（zhen-qiyun），xianzhi 中删除
- `合欢宗秘辛` 中与其他类别重叠的条目，合并或标记为引用

## 风险评估与依赖

| 风险 | 等级 | 应对 |
|---|---|---|
| 去重过度：某条目在游戏实际剧情中有重要作用 | 中 | 保留所有不重复的条目，只合并完全相同或高度相似的 |
| 新增条目质量不足 | 中 | 参照现有条目的描述风格和效果格式 |
| 删除的条目被运行时引用 | 低 | 气运/天赋/背景通过名称引用，不通过代码引用 |
| 测试覆盖 | 低 | 项目无测试框架，通过 lint + build 验证 |
