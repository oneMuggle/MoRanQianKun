# 现代纪元故事模块管理方案

## 1. 背景与目标

### 1.1 当前问题

现代纪元（`contemporary_urban`）目前有两个 NSFW 模块：`campusNSFW` 和 `urbanDriverNSFW`。每个新模块需要在 6+ 个独立文件中手动添加代码：

| 文件 | 修改内容 |
|------|---------|
| `models/game-settings.ts` | 添加 `设置类型` 和导入 |
| `models/{module}/index.ts` | 定义模块设置接口和默认值 |
| `utils/gameSettings.ts` | 添加设置规范化逻辑 |
| `prompts/runtime/nsfw.ts` | 添加模块提示词构建调用 |
| `prompts/runtime/{module}.ts` | 定义模块提示词构建函数 |
| `hooks/useGame/mainStoryRequest.ts` | 添加运行时参数类型和传递 |
| `hooks/useGame/sendWorkflow/index.ts` | 添加参数构建和传递 |
| `hooks/useGame/{module}Integration.ts` | 添加参数提取器 |
| `components/features/Settings/SettingsPanel.tsx` | 添加设置页签和渲染 |
| `components/features/Settings/{Module}Settings.tsx` | 定义设置 UI 组件 |

按照 `docs/plans/现代纪元NSFW模块扩展方案.md` 中规划的 28+ 模块，如果沿用当前模式，将产生 **168+ 次手动修改**，维护成本不可接受。

### 1.2 目标

1. **集中注册**：单一注册点可查看所有现代纪元模块
2. **独立激活**：每个模块通过 `gameConfig` 独立启用/禁用
3. **提示词动态组装**：活跃模块在运行时贡献提示词片段
4. **状态延迟初始化**：仅在模块激活时初始化
5. **跨模块链接**：支持 NPC 跨模块、后果跨模块
6. **向后兼容**：保持现有 campusNSFW 和 urbanDriver 模块功能不变
7. **可扩展性**：新增模块只需添加 1-2 个文件 + 1 行注册

## 2. 现状分析

### 2.1 现有模块集成路径

```
SettingsPanel.tsx → CampusNSFWSettings.tsx (UI)
                    ↓ 保存
game-settings.ts → 校园NSFW设置 (类型定义)
                    ↓ 规范化
gameSettings.ts → 规范化游戏设置 (默认值填充)
                    ↓ 传递
useGame.ts → 校园系统状态 (状态管理)
                    ↓ 提取
bdsmStateIntegration.ts → 构建校园NSFW参数 (参数构建)
                    ↓ 传递
sendWorkflow/index.ts → 校园NSFW参数
                    ↓ 传递
mainStoryRequest.ts → 校园NSFW参数
                    ↓ 调用
prompts/runtime/nsfw.ts → 构建运行时NSFW提示词
                              ↓ 调用
                          prompts/runtime/campusNSFW.ts → 构建校园NSFW完整叙事约束
```

### 2.2 关键发现

1. `utils/gameSettings.ts` 的 `规范化游戏设置` 仅处理 `校园NSFW设置`，**未处理** `都市网约车NSFW设置`，这是现有不一致性
2. `prompts/runtime/nsfw.ts` 的 `构建运行时NSFW提示词` 硬编码了两个模块的 if 分支
3. `功能附加块是否启用` 在 `promptFeatureToggles.ts` 中管理修炼/生存/里武侠等功能开关，但**不覆盖 NSFW 模块**
4. 设置面板中的页签是硬编码的数组

### 2.3 可复用模式

- `功能附加块` 的 `<!-- PROMPT_FEATURE:xxx:START -->` 标记模式可用于模块提示词隔离
- `规范化游戏设置` 的读取布尔/读取文本模式可复用
- `默认校园NSFW设置` 的对象字面量模式可复用

## 3. 架构设计

### 3.1 总体架构

```
故事模块注册表 (StoryModuleRegistry)
       │
       ├─ 注册模块 (静态)
       │
       ├─ 获取活跃模块 (运行时, 基于 gameConfig)
       │
       ▼
模块编排器 (ModuleOrchestrator)
       │
       ├─ 初始化模块状态 (延迟)
       ├─ 构建模块参数 (运行时提取)
       └─ 组装模块提示词 (动态拼接)
       │
       ▼
主剧情请求 (mainStoryRequest)
       │
       └─ 构建运行时额外提示词 ← 接收所有模块参数
```

### 3.2 核心接口定义

```typescript
// utils/storyModule/types.ts

/**
 * 故事模块通用接口
 * @param Settings - 模块设置类型（如 校园NSFW设置）
 * @param State - 模块运行时状态类型
 * @param PromptParams - 传递给提示词的运行时参数类型
 */
export interface StoryModule<
  Settings = Record<string, unknown>,
  State = Record<string, unknown>,
  PromptParams = Record<string, unknown>
> {
  /** 模块唯一标识，如 'campus_nsfw', 'urban_driver_nsfw' */
  id: string;

  /** 模块显示名称，如 '校园NSFW' */
  name: string;

  /** 所属时代 ID，如 'contemporary_urban' */
  eraId: string;

  /** 语义化版本 */
  version: string;

  /** 优先级（数字越大越先处理），用于提示词组装顺序 */
  priority: number;

  /** 模块分类 */
  category: 'nsfw' | 'gameplay' | 'narrative' | 'social';

  /** 简短描述 */
  description: string;

  /** 主开关在 gameConfig 中的键名，如 '启用校园NSFW' */
  masterToggleKey: string;

  /** 依赖的其他模块 ID（为空表示无依赖） */
  dependencies: string[];

  /** 设置类型的默认值 */
  defaultSettings: Settings;

  /**
   * 规范化设置：处理用户输入的类型转换和校验
   */
  normalizeSettings: (raw: Partial<Settings>) => Settings;

  /**
   * 从游戏运行时状态中提取本模块的提示词参数
   */
  extractPromptParams: (
    gameState: 游戏状态快照,
    settings: Settings
  ) => PromptParams | null;

  /**
   * 构建本模块的提示词片段
   */
  buildPromptFragment: (
    params: PromptParams,
    settings: Settings
  ) => string;

  /**
   * 可选：AI 响应中的状态更新标签
   */
  responseTag?: string;
}

/** 游戏状态快照：extractPromptParams 需要的上下文 */
export interface 游戏状态快照 {
  eraId: string;
  角色状态: Record<string, unknown>;
  环境状态: Record<string, unknown>;
  社交状态: Record<string, unknown>;
  校园系统?: Record<string, unknown>;
  网约车系统?: Record<string, unknown>;
  [key: string]: unknown;
}
```

### 3.3 模块注册表

```typescript
// utils/storyModule/registry.ts

import type { StoryModule } from './types';

export class 故事模块注册表 {
  private static _模块映射 = new Map<string, StoryModule>();

  static 注册模块(module: StoryModule): void {
    if (this._模块映射.has(module.id)) {
      console.warn(`[故事模块] 模块 ${module.id} 已被注册，将覆盖`);
    }
    this._模块映射.set(module.id, module);
  }

  static 获取模块(id: string): StoryModule | undefined {
    return this._模块映射.get(id);
  }

  /** 获取指定时代的所有模块（按优先级排序） */
  static 获取时代模块(eraId: string): StoryModule[] {
    return Array.from(this._模块映射.values())
      .filter(m => m.eraId === eraId)
      .sort((a, b) => b.priority - a.priority);
  }

  /** 获取当前活跃模块（基于 gameConfig 的主开关） */
  static 获取活跃模块(
    eraId: string,
    gameConfig: Record<string, unknown>
  ): StoryModule[] {
    return this.获取时代模块(eraId).filter(module => {
      const masterValue = gameConfig[module.masterToggleKey];
      if (masterValue === undefined) {
        return true; // 默认启用
      }
      return masterValue === true;
    });
  }

  /** 检查模块依赖是否满足 */
  static 依赖是否满足(moduleId: string, activeModuleIds: Set<string>): boolean {
    const module = this.获取模块(moduleId);
    if (!module) return false;
    return module.dependencies.every(dep => activeModuleIds.has(dep));
  }

  static 获取所有模块ID(): string[] {
    return Array.from(this._模块映射.keys());
  }

  static 获取模块摘要(): Array<{
    id: string; name: string; eraId: string; version: string;
    category: string; priority: number; dependencies: string[];
  }> {
    return Array.from(this._模块映射.values()).map(m => ({
      id: m.id, name: m.name, eraId: m.eraId, version: m.version,
      category: m.category, priority: m.priority, dependencies: m.dependencies
    }));
  }
}
```

### 3.4 模块编排器

```typescript
// utils/storyModule/orchestrator.ts

import type { StoryModule, 游戏状态快照 } from './types';
import { 故事模块注册表 } from './registry';

export interface 模块运行时上下文 {
  activeModules: StoryModule[];
  moduleSettings: Map<string, Record<string, unknown>>;
  moduleParams: Map<string, Record<string, unknown>>;
}

export function 初始化模块编排(
  eraId: string,
  gameConfig: Record<string, unknown>
): 模块运行时上下文 {
  const activeModules = 故事模块注册表.获取活跃模块(eraId, gameConfig);

  const activeIds = new Set(activeModules.map(m => m.id));
  const satisfiedModules = activeModules.filter(m =>
    故事模块注册表.依赖是否满足(m.id, activeIds)
  );

  const moduleSettings = new Map<string, Record<string, unknown>>();
  for (const module of satisfiedModules) {
    const rawSettings = (gameConfig as any)[`${module.id}设置`];
    const settings = module.normalizeSettings(rawSettings ?? {});
    moduleSettings.set(module.id, settings);
  }

  return {
    activeModules: satisfiedModules,
    moduleSettings,
    moduleParams: new Map(),
  };
}

export function 提取模块参数(
  context: 模块运行时上下文,
  gameState: 游戏状态快照
): void {
  for (const module of context.activeModules) {
    const settings = context.moduleSettings.get(module.id)!;
    const params = module.extractPromptParams(gameState, settings);
    if (params !== null) {
      context.moduleParams.set(module.id, params);
    }
  }
}

export function 构建故事模块提示词(
  context: 模块运行时上下文
): string {
  const fragments: string[] = [];

  for (const module of context.activeModules) {
    const settings = context.moduleSettings.get(module.id)!;
    const params = context.moduleParams.get(module.id);
    if (!params) continue;

    const fragment = module.buildPromptFragment(
      params as Record<string, unknown>,
      settings
    );

    if (fragment) {
      fragments.push(
        `<!-- PROMPT_MODULE:${module.id}:START -->\n` +
        fragment +
        `\n<!-- PROMPT_MODULE:${module.id}:END -->`
      );
    }
  }

  return fragments.join('\n\n');
}
```

## 6. 提示词动态组装

修改 `prompts/runtime/nsfw.ts` 的 `构建运行时NSFW提示词`：

```typescript
// prompts/runtime/nsfw.ts

import { 构建故事模块提示词 } from '../../utils/storyModule/orchestrator';

const 构建运行时NSFW提示词 = (
  customPrompt: string,
  baseNsfwPrompt: string,
  options?: {
    启用NSFW模式?: boolean;
    nsfw场景类型?: NSFW场景类型;
    时代配置ID?: string;
    故事模块上下文?: 模块运行时上下文;
    校园NSFW参数?: { ... };
    都市网约车NSFW参数?: { ... };
  }
): string => {
  const custom = typeof customPrompt === 'string' ? customPrompt.trim() : '';
  const nsfwEnabled = options?.启用NSFW模式 === true;

  if (!nsfwEnabled) return custom;

  const 组件: string[] = [baseNsfwPrompt];

  // 1. 时代叙事约束（保持不变）
  const lixiangConstraint = 自动选择叙事约束(
    options?.时代配置ID,
    options?.nsfw场景类型 || '完全展开'
  );
  if (lixiangConstraint) 组件.push(lixiangConstraint);

  // 2. 新路径：动态组装所有故事模块提示词
  if (options?.故事模块上下文) {
    const modulePrompt = 构建故事模块提示词(options.故事模块上下文);
    if (modulePrompt) 组件.push(modulePrompt);
  }

  // 3. 旧路径：兼容期保留
  if (!options?.故事模块上下文) {
    if (options?.校园NSFW参数) {
      const campusConstraint = 构建校园NSFW叙事约束(options.校园NSFW参数);
      if (campusConstraint) 组件.push(campusConstraint);
    }
    if (options?.都市网约车NSFW参数) {
      const driverConstraint = 构建都市网约车完整叙事约束(options.都市网约车NSFW参数);
      if (driverConstraint) 组件.push(driverConstraint);
    }
  }

  组件.push(custom);
  return 组件.filter(Boolean).join('\n\n').trim();
};
```

## 7. 与现有模块兼容方案

### 7.1 三阶段迁移策略

| 阶段 | 策略 | 说明 |
|------|------|------|
| **阶段 1：并行运行** | 新注册表 + 旧路径同时运行 | 注册表注册 campus/driver，旧路径仍然生效。新路径仅作为验证 |
| **阶段 2：新路径优先** | 新路径优先，旧路径回退 | `mainStoryRequest.ts` 开始传递 `故事模块上下文` |
| **阶段 3：清理旧路径** | 移除旧参数和分支 | 删除旧参数类型和传递逻辑，删除独立 integration 文件 |

## 11. 设计决策说明

### 11.1 为什么不使用 Zustand？

模块注册表是纯静态注册（Map），编排器是运行时计算，不涉及跨组件状态同步。引入 Zustand 会增加不必要的复杂度。

### 11.2 为什么保留旧参数传递？

三阶段迁移策略确保零风险。旧参数传递在过渡期充当安全网。

### 11.3 为什么使用中文标识符？

遵循项目现有命名约定。

### 11.4 为什么使用 `PROMPT_MODULE` 标记而非 `PROMPT_FEATURE`？

`PROMPT_FEATURE` 已有特定语义（修炼/生存/里武侠等）。新模块使用 `PROMPT_MODULE` 区分，避免冲突。两者可共存。
