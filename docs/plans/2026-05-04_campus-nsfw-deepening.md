# 实施方案：校园纪元 NSFW 深化

> 创建日期：2026-05-04
> 最后更新：2026-05-04 (v1.4 - 校园祭 NSFW 深化)
> 状态：实施中（Phase 1-4 已完成，Phase 5 集成中）

---

## 概述

为"现代校园"时代（contemporary_campus）深化 NSFW 内容与核心玩法的集成。通过引入欲望状态机、关系轨道系统、后果系统、手机 NSFW App 和成长曲线，使 NSFW 内容不再是单纯的叙事生成，而是嵌入探索、社交、决策、成长的核心循环中。

---

## 需求分析

### 当前状态
- NSFW 内容主要依赖 AI prompt 生成，缺乏游戏状态驱动
- 好感度系统是唯一的关系度量，无法区分情感和肉体关系
- 没有后果机制，NSFW 行为不影响游戏世界
- 缺少可视化的进度跟踪工具

### 目标
- NSFW 内容与核心玩法循环（探索、社交、战斗、成长）深度绑定
- 引入欲望状态机和关系轨道，区分情感和肉体关系
- 后果系统让 NSFW 行为产生真实的校园影响
- 提供细粒度的玩家设置控制

### 范围
- 仅影响校园时代（contemporary_campus），不影响武侠/古代时代
- 所有新增功能可通过设置完全关闭
- 旧存档完全兼容

### 深化方向（v1.1 新增）

本计划在 v1.0 基础之上，向以下方向深化设计：

- **露出系统** — 从被动风险到主动追求的渐进式暴露玩法
- **公开隐秘侵犯** — 公共场所的隐蔽互动，旁观者可能察觉但 NPC 保持表面正常
- **旁观者系统** — 周围人群的意识觉醒和反应机制
- **羞耻感机制** — NPC 在暴露/公开场景下的心理博弈
- **校园活动事件** — 运动会、开学典礼等大型场合中的 NSFW 事件
- **SM/支配服从** — 权力天平、服从度、契约关系的动态 SM 系统
- **桌游社交 NSFW** — 密室逃脱/狼人杀/剧本杀/派对游戏中的自然 NSFW 融合
- **校园祭** — 筹备/举办/后夜祭完整叙事弧线，5 种主题，鬼屋/摊位/告白场景

## 详细功能设计

### 功能 1: 欲望状态机

**核心概念**: 每个 NPC 有独立的欲望状态，不受好感度直接控制。好感度高不代表欲望阶段高。

#### 阶段定义

| 阶段 | 行为特征 | 可解锁互动 | 推进条件 |
|------|---------|-----------|---------|
| 克制 | 主动保持距离，回避身体接触 | 眼神交流、递笔记 | 多次正面互动 |
| 试探 | 开始寻找独处机会，制造接触 | 牵手、靠肩膀、摸头 | 选择激进选项 |
| 渴望 | 主动创造独处场景 | 拥抱、轻吻、抚摸 | 欲望阶段进度满 |
| 沉沦 | 不再在意暴露风险 | 完全亲密 | 多次亲密互动 |
| 支配 | 主导关系走向，设定规则 | 所有互动 + 特殊事件 | 关系轨道推进 |

#### 推进公式

```
进度增加 = 基础值(互动类型) x 地点加成 x 选择系数 x 随机波动(+/-10%)

其中：
- 基础值：眼神交流=5, 牵手=10, 拥抱=15, 轻吻=20, 深吻=25, 抚摸=30, 完全亲密=40
- 地点加成：极低=0.5, 低=0.8, 中=1.0, 高=1.3, 极高=1.5
- 选择系数：保守=0.6, 中性=1.0, 激进=1.5
```

#### 冷却机制

```
互动冷却期 = 基础冷却(互动类型) x 欲望阶段修正
- 眼神交流：0 回合
- 牵手/拥抱：1 回合
- 亲吻/抚摸：2 回合
- 完全亲密：3 回合
- 欲望阶段"沉沦"以上：冷却减半
```

### 功能 2: 关系轨道系统

**核心概念**: 关系轨道描述玩家与 NPC 的关系性质，由玩家的累计选择决定。

#### 轨道定义

| 轨道 | 特征 | 解锁条件 | 特殊事件 |
|------|------|---------|---------|
| 纯爱 | 情感驱动，缓慢推进 | 默认 | 告白事件、毕业承诺 |
| 暧昧 | 若即若离，不确定性 | 多次选择中性选项 | 吃醋事件、试探底线 |
| 肉体 | 身体驱动，快速推进 | 多次选择激进选项 | 偷情事件、欲望失控 |
| 支配 | 权力不对等 | 欲望阶段达到"支配" | 服从测试、权力交换 |
| 多角 | 同时与多人发展 | 启用多角设置 | 修罗场、摊牌事件 |

#### 轨道判定算法

```
每 5 个里程碑评估一次：
- 保守选项占比 > 60% -> 纯爱
- 中性选项占比 > 60% -> 暧昧
- 激进选项占比 > 60% -> 肉体
- 欲望阶段 = "支配" -> 支配
- 同时 2+ NPC 欲望阶段 >= "渴望" -> 多角
```

### 功能 3: 后果系统

**核心概念**: NSFW 行为不是无代价的，有真实的校园后果。

#### 后果类型

| 类型 | 触发条件 | 影响 | 解决方式 |
|------|---------|------|---------|
| 暴露风险 | 高暴露风险地点互动 | 临时属性下降 | 等待冷却 |
| 流言传播 | 暴露风险累积到阈值 | NPC 态度变化 | 公开/隐瞒选择 |
| 学业下滑 | 过多时间花在 NSFW | 考试失败、导师警告 | 临时补习 |
| 多角冲突 | 同时与 2+ NPC 发展 | 关系危机 | 选择站队 |
| 心理负担 | 与欲望阶段不匹配的推进 | NPC 回避、好感度下降 | 道歉/沟通 |
| 毕业危机 | 学业影响达到"显著" | 可能无法毕业 | 紧急补救 |

#### 流言传播模拟

```
流言等级 (0-5)：
- 0: 无人知晓
- 1: 闺蜜/好友察觉
- 2: 小范围窃窃私语
- 3: 班级内传播
- 4: 年级内知晓
- 5: 全校皆知

增长公式：
新等级 = min(5, 当前等级 + floor(暴露事件数 / 3))
衰减：每 10 回合无新暴露事件，等级 -1
```

### 功能 5.5: 露出系统（新增深化）

**核心概念**: 露出从"被动风险"转变为"主动追求的快感"。NPC 随着欲望阶段推进，逐渐从害怕暴露变为享受暴露。

#### 露出偏好等级

| 等级 | 名称 | 行为特征 | 解锁场景 |
|------|------|---------|---------|
| 0 | 抗拒 | 极度害怕被发现，主动避开人群 | 无 |
| 1 | 紧张 | 担心暴露但在私密半公开场所可接受 | 更衣室空无一人时 |
| 2 | 犹豫 | 被发现会紧张但也有一丝兴奋 | 窗帘拉开一半的房间 |
| 3 | 享受 | 开始享受被偷看的刺激感 | 走廊角落有人经过时 |
| 4 | 主动 | 主动寻找可能被发现的机会 | 操场边、教学楼走廊 |
| 5 | 追求 | 以被更多人看到为终极快感 | 校园公开活动场合 |

#### 露出偏好推进公式

```
露出偏好等级推进 = 基础触发概率 x 修正系数

基础触发概率（每次在公共/半公共场所互动时）：
- 欲望阶段"克制"/"试探": 5%
- 欲望阶段"渴望": 15%
- 欲望阶段"沉沦": 30%
- 欲望阶段"支配": 50%

修正系数：
- 玩家选择"冒险暴露"选项: x2.0
- 玩家选择"安全隐蔽"选项: x0.3
- 当前暴露风险值 > 70: x1.5
- NPC 性格标签"外向": x1.3
- NPC 性格标签"内向": x0.7
- 之前发生过暴露事件: x1.8
```

#### 露出场景阶梯

```
阶梯 1: 半私密场所（窗帘后、门后）
  -> NPC 动作：紧张地拉上窗帘、确认门锁
  -> 成功判定：未被发现，获得"安全快感"加成

阶梯 2: 半公共场所（走廊尽头、楼梯转角）
  -> NPC 动作：时不时查看是否有人经过
  -> 成功判定：可能被发现但未发现，NPC 露出偏好+1

阶梯 3: 公共场所（教室、图书馆座位下）
  -> NPC 动作：表面正常，桌下/裙下秘密互动
  -> 成功判定：周围人未察觉或假装未察觉
  -> 失败判定：被同学/老师发现 -> 触发"暴露事件"

阶梯 4: 公开展示（运动后更衣、舞台表演）
  -> NPC 动作：在可能被围观的情况下展示
  -> 需要露出偏好等级 >= 4
  -> 成功判定：获得"暴露快感"大幅推进欲望

阶梯 5: 公开场合的隐秘侵犯（活动会场、典礼中）
  -> NPC 动作：在大量人群中进行秘密互动
  -> 需要露出偏好等级 = 5 + 欲望阶段"沉沦"以上
  -> 旁观者察觉概率独立计算
```

#### 暴露事件处理流程

```
1. 判定是否被发现（基于地点暴露风险 + 时间段 + 流言等级）
2. 如果被发现有三个分支：
   a. 被发现者默默走开（假装没看到）-> 流言等级 +1
   b. 被发现者震惊/举报 -> 触发"严重暴露事件"后果
   c. 被发现者偷拍/传播 -> 流言等级 +2 + 新增"网络传播"后果
3. NPC 反应取决于露出偏好等级：
   - 等级 0-1: 恐慌、羞耻、可能要求断绝关系
   - 等级 2-3: 紧张但兴奋，可能加速推进
   - 等级 4-5: 享受刺激，主动要求继续
```

### 功能 5.6: 公开隐秘侵犯系统（新增深化）

| 场景类型 | 描述 | 紧张度 | 发现概率 |

| 桌下互动 | 教室/图书馆桌面下的秘密接触 | 中 | 25% |

| 车内密闭 | 校车/私家车后座的秘密互动 | 低 | 10% |

#### 紧张度系统

```
紧张度 = f(场景基础值, 周围人数, 互动强度, 暴露风险值, NPC 露出偏好)

紧张度范围: 0-100

周围人数修正：
- 0-2 人: x0.5
- 3-10 人: x1.0
- 11-30 人: x1.5
- 30+ 人: x2.0

互动强度修正：
- 牵手/抚摸腿部: 基础值 x0.5
- 更深入的接触: 基础值 x1.5
- 完全亲密: 基础值 x2.5

NPC 状态修正：
- NPC 正在公开发言: x2.0 (极度紧张)
- NPC 需要回应他人: x1.5
- NPC 可以保持沉默: x1.0
```

#### 紧张度影响

```
紧张度 0-30:

- 成功增加欲望阶段进度 x1.2

- 发现概率 +30%
- 成功增加欲望阶段进度 x1.5

- NPC 极可能被察觉异常

- 成功增加欲望阶段进度 x2.0

### 功能 5.7: 旁观者与流言深化系统（新增深化）

**核心概念**: 深化原有流言系统，加入"发现-传播-发酵"的完整链条，以及网络时代的特殊传播机制。

#### 流言传播链条

```
发现阶段 -> 传播阶段 -> 发酵阶段 -> 后果阶段

发现阶段：
- 目击者数量（1人、2-3人、更多人）
- 发现方式（直接看到、听到声音、偷拍证据）

传播阶段：
- 口头传播（窃窃私语 -> 小范围 -> 大范围）
- 网络传播（匿名论坛、社交群、截图分享）
- 速度取决于"校园规模"和"事件严重程度"

发酵阶段：
- 事件被添油加醋（严重程度升级）
- 产生不同版本的传言
- 可能引发校园关注/调查

后果阶段：
- NPC 被约谈/警告
- 校园舆论压力
- 关系状态被迫公开化
```

#### 网络传播机制

```
网络流言等级 (0-5)：
- 0: 无网络痕迹
- 1: 匿名贴吧出现暗示性帖子
- 2: 校园群出现讨论
- 3: 截图/证据在校园内流传
- 4: 外校论坛也有讨论
- 5: 登上热搜/被媒体报道

网络传播触发条件：
- 有偷拍证据 + 暴露事件 -> 必触发网络传播
- 无证据但流言等级 >= 3 -> 可能触发

网络传播衰减：

### 功能 5.9: SM/支配服从系统（新增深化）

**核心概念**: SM 不是单纯的"暴力"，而是一种权力交换的动态关系。在校园背景下，SM 元素以心理支配、服从测试、角色扮演等温和形式出现，逐渐深入到更明确的身体控制。核心体验是"信任"和"控制权的让渡"。

## 风险评估与应对

### Risk 1: 核心文件修改风险 (High)
- **描述**: `hooks/useGame.ts` 是核心文件，改动可能引入回归 bug
- **应对**:
  - NSFW 逻辑封装在独立引擎文件中
  - 在 `useGame.ts` 中只做函数调用，不嵌入复杂逻辑
  - 每个改动点做独立验证

### Risk 2: Prompt 调优不确定性 (Medium)
- **描述**: AI 对 NSFW 约束的理解可能不稳定
- **应对**:
  - 使用结构化 prompt（标签、分点、明确规则）
  - 提供大量具体场景示例
  - 预留 prompt 热更新能力

### Risk 3: 多 NPC 并发复杂度 (Medium)
- **描述**: 同时与多个 NPC 发展 NSFW 关系的组合爆炸
- **应对**:
  - 限制同时发展的 NPC 数量（默认 3，可配置）
  - 多角冲突事件作为自然调节机制
  - 每个 NPC 独立状态，不互相干扰除非触发冲突

### Risk 4: 存档兼容性 (Medium)
- **描述**: 旧存档加载时新字段缺失
- **应对**:
  - 所有新增字段使用可选类型
  - 加载时提供默认值
  - schema version 检查

### Risk 5: UI 布局冲突 (Low)
- **描述**: 新增 Tab 可能与现有布局冲突
- **应对**:
  - 复用现有 Tab 组件模式
  - 桌面端/移动端分别实现
  - 预留滚动和折叠机制

### Risk 6: 敏感内容平衡 (High) - v1.1 新增
- **描述**: 露出和公开隐秘侵犯涉及敏感主题，需要确保玩家完全知情同意且可随时关闭
- **应对**:
  - 所有功能默认关闭，需主动开启
  - 独立的开关控制每个子系统
  - Prompt 中明确叙事边界，避免过度描写
  - 提供内容预警选项

### Risk 7: 旁观者系统复杂度 (Medium) - v1.1 新增
- **描述**: 每个 NPC 都可能有独立的旁观者反应，状态爆炸
- **应对**:
  - 旁观者使用概率模型而非逐个模拟
  - 每回合只抽样检查 1-2 个关键旁观者
  - 旁观者状态用完即销毁，不持久化所有细节

### Risk 8: 校园活动同步 (Low) - v1.1 新增
- **描述**: 校园活动需要与游戏时间线同步
- **应对**:
  - 活动日历使用配置驱动，与游戏回合关联
  - 活动状态只记录当前进行中活动，不维护全部历史

### Risk 9: SM 内容敏感性 (Critical) - v1.2 新增
- **描述**: SM 内容比普通 NSFW 更敏感，需要更严格的同意机制和边界控制
- **应对**:
  - SM 系统完全独立开关，默认关闭
  - 权力天平系统确保"双方同意"是前提
  - Prompt 约束强调心理描写优先，避免过度身体描写
  - "安全词"机制在 prompt 中体现，NPC 可随时退出
  - 内容强度分级允许玩家选择只体验心理层面

### Risk 10: 状态爆炸 (Medium) - v1.2 新增
- **描述**: SM 系统引入权力天平、服从度、契约、指令队列等多层状态
- **应对**:
  - 指令队列限制最大 3 条，过期自动清除
  - SM 场景历史只保留最近 20 条
  - 契约只支持单一契约（新契约覆盖旧契约）
  - 权力天平和服从度是标量值，存储开销极小

### Risk 11: 桌游类型复杂度 (High) - v1.3 新增
- **描述**: 5 种桌游各有独立的 NSFW 子系统，状态机和场景生成逻辑差异大
- **应对**:
  - 每种桌游的 NSFW 逻辑封装在独立函数中
  - 共用 `桌游状态` 接口，内部用联合类型区分
  - 优先实现密室逃脱（最核心的独处机制），其余按优先级渐进

### Risk 12: AI Prompt 桌游融合 (Medium) - v1.3 新增
- **描述**: AI 需要同时理解游戏规则和 NSFW 场景，生成自然融合的内容
- **应对**:
  - 每种桌游提供独立的 prompt 约束模板
  - prompt 中包含游戏规则简述，确保 AI 理解上下文
  - 提供具体场景示例，如"黑暗中牵手"的描写范式

### Risk 13: 校园祭叙事长度 (High) - v1.4 新增
- **描述**: 完整校园祭跨越 20-33 回合，叙事弧线太长可能导致中间拖沓
- **应对**:
  - 校园祭分阶段执行，每阶段独立可验证
  - 玩家可选择跳过某些阶段（如直接到举办期）
  - 每个阶段至少有一个核心 NSFW 场景，不依赖数量

### Risk 14: 校园祭与桌游系统重叠 (Medium) - v1.4 新增
- **描述**: 校园祭中的鬼屋/摊位/派对可能与桌游系统的场景重叠
- **应对**:
  - 校园祭场景优先使用专属逻辑
  - 如果校园祭期间触发桌游（如派对游戏），调用桌游子系统
  - 两个系统通过状态标记避免重复触发
