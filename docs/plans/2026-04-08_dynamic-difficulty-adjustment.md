# 动态难度调整 (Dynamic Difficulty Adjustment)

## 概述

根据玩家表现动态调整游戏难度，提供更平衡的游戏体验。

## 需求分析

### 当前状态
- 游戏已有 5 个难度等级: relaxed / easy / normal / hard / extreme
- 难度在开局时设置，通过 `世界设置.difficulty` 保存
- 难度影响战斗倍率、资源经济、生存压力、社交世界、失败代价
- 当前难度固定，不随玩家表现变化

### 需要解决的问题
1. 新手可能因难度过高而流失（hard/extreme）
2. 熟练玩家可能因难度过低而无聊（relaxed/easy）
3. 缺乏客观指标来评估玩家实力
4. 缺乏根据表现自动调整难度的机制

## 设计方案

### 1. 游戏统计模型

```typescript
// models/system.ts 新增
interface 游戏统计 {
  总回合数: number;
  战斗次数: number;
  胜利次数: number;
  失败次数: number;
  死亡次数: number;
  任务完成次数: number;
  任务失败次数: number;
  跨境挑战次数: number; // 越级挑战
  跨境成功次数: number;
  濒死次数: number; // 单次回合内精力<=10%或部位重伤
  连续失败次数: number; // 追踪连续失败用于风险预警
  难度调整历史: 难度调整记录[];
}

interface 难度调整记录 {
  时间: number;
  原难度: 游戏难度;
  新难度: 游戏难度;
  原因: string;
  玩家表现: {
    胜率: number;
    死亡次数: number;
    任务完成率: number;
  };
}
```

### 2. 难度评估指标

| 指标 | 计算方式 | 权重 |
|------|----------|------|
| 战斗胜率 | 胜利次数 / 战斗次数 | 30% |
| 任务完成率 | 任务完成 / (任务完成+任务失败) | 20% |
| 生存指数 | 1 - (死亡次数 / max(总回合数/10, 1)) | 25% |
| 越级成功率 | 跨境成功次数 / 跨境挑战次数 | 15% |
| 综合评级 | 加权求和后映射到 0-100 分 | 100% |

### 3. 动态调整规则

```typescript
const 难度评估阈值 = {
  // 降难度条件 (玩家表现差)
  降难度: {
    胜率低于: 0.3,      // 战斗胜率 < 30%
    死亡次数超过: 3,    // 5场内死亡 > 3次
    连续失败超过: 3,    // 连续失败 > 3次
  },
  // 升难度条件 (玩家表现好)
  升难度: {
    胜率高于: 0.7,      // 战斗胜率 > 70%
    任务完成率高于: 0.8, // 任务完成率 > 80%
    越级成功率高: 0.6,   // 越级成功率 > 60%
  },
  // 调整冷却
  冷却回合数: 10, // 至少10回合后才能再次调整
  升降幅度: 1,   // 每次最多调整1级
};
```

### 4. 调整触发时机

- 每次回合结束时评估
- 仅在 非战斗/安全场景 时触发调整
- 调整后自动记录到难度调整历史

### 5. 难度等级映射

```
综合评级 0-20  → relaxed (轻松)
综合评级 21-40 → easy (简单)
综合评级 41-60 → normal (正常)
综合评级 61-80 → hard (困难)
综合评级 81-100 → extreme (极限)
```
