# 糖爹/糖宝关系 NSFW 模块详细方案

## 一、概述

### 1.1 模块定位

糖爹/糖宝关系系统是现代纪元 NSFW 模块的 P0 核心模块，聚焦于物质与情感不对等交换关系中的权力动态、心理博弈和社会议题。

### 1.2 核心特色

**与传统游戏差异化：**
- 关系是"精心计算的真诚"——表面是感情，实际是合同，但过程中可能产生真实情感
- 权力不对等是结构性的——年龄差、财富差、社会经验差形成系统性优势
- 关系边界模糊——不是简单的"给钱办事"，存在大量灰色地带
- 身份认知的挣扎——"我是被包养的"还是"我正在谈恋爱"？
- 曝光风险是悬在头顶的剑——一旦暴露，生活可能崩塌

### 1.3 题材敏感度

```
本模块涉及：
✓ 物质与情感的交换关系
✓ 拜金主义与自我认同冲突
✓ 权力不对等关系（年龄、财富、社会地位）
✓ 关系的隐匿性与曝光风险
✓ 多重关系的同时维持
✓ 关系结束后的心理落差
✓ 原配/家人的发现与冲突

本模块不包括：
✗ 任何实际性行为的具体描写
✗ 未成年人相关敏感内容
✗ 真实人物/平台名称
```

### 4.2 关系建立机制

**初次见面流程：**

```
1. 互相了解（线上）
   ├── 查看照片/资料
   ├── 试探性聊天
   └── 交换基本期望

2. 条件谈判
   ├── 糖宝提出期望（零花钱、见面频率等）
   ├── 糖爹提出要求（专属、排他性等）
   └── 双方协商一致

3. 第一次见面
   ├── 选择公共场所（高档餐厅/酒店）
   ├── 观察对方真人
   ├── 决定是否继续

4. 关系确定
   ├── 进入蜜月期或筛选下一位
   └── 开始物质往来
```

**关系谈判要素：**

| 要素 | 糖宝关心 | 糖爹关心 |
|------|----------|----------|
| 零花钱 | 金额大小 | 给多少能达到目的 |
| 见面频率 | 时间安排 | 能获得多少陪伴 |
| 排他性 | 自由程度 | 是否专属 |
| 礼物 | 奢侈品价值 | 礼物的意义 |
| 曝光风险 | 隐私保护 | 会不会被拍 |
| 情感需求 | 有没有被在乎 | 有没有真情实感 |
| 未来发展 | 这段关系的终点 | 她会不会纠缠 |

### 4.5 危机触发机制

```typescript
interface 曝光风险评估 {
```

### 5.1 初次见面场景

**场景一：高档餐厅初见**

```
触发条件：
- 糖宝与糖爹在线上谈判后决定见面
- 选择高档餐厅作为见面地点

场景描述：
[米其林餐厅 - 晚7点]
第一次线下见面...

关键要素：
- 见面前的紧张与期待
- 真人vs照片的落差
- 糖爹的气场、谈吐、穿着
- 糖宝的穿着、礼仪、表现
- 话题试探（了解彼此更多）
- 观察对方真实意图

选择点：
- 穿着打扮（低调vs展示）
- 话题应对（主动vs被动）
- 消费态度（放不开vs自然）
- 是否接受饭后安排（回家/继续）
```

**场景二：干爹圈聚会**

```
触发条件：
- 糖爹带糖宝参加圈内聚会
- 糖宝第一次接触"糖圈"

场景描述：
[私人会所 - 晚9点]
干爹们带着各自的糖宝聚会...

聚会内容：
- 糖爹们的商务交流
- 糖宝们的社交暗战
- 礼物炫耀/比较
- 圈子规矩的观察

糖宝的处境：
- 认识其他糖宝
- 被比较、被打量
- 听到圈子里的故事
- 了解"行情"
- 意识到自己的位置
```

### 5.2 日常约会场景

**场景三：奢侈品购物**

```
触发条件：
- 糖宝获得购物预算
- 跟随糖爹或独自购物

场景描述：
[顶级商场 - 下午]
买买买的日常...

购物类型：
- 指定礼物（糖爹明确要送）
- 任选礼物（给额度自己选）
- 撒娇要礼物（糖宝主动要求）
- 必需品采购（日常开销）

关键选择：
- 选什么价位的？
- 是否超出糖爹预期？
- 如何表达感谢？
- 是否顺便给糖爹买东西？
```


### 5.3 危机场景

**场景五：原配找上门**

```
触发条件：
- 原配发现了糖宝的存在
- 决定直接对峙

场景描述：
[某场所 - 突然]
原配带着证据出现...

原配反应类型：
- 愤怒攻击型：直接辱骂、动手
- 冷静谈判型：要求给钱离开
- 威胁曝光型：以公开威胁
- 收集证据型：先取证再行动
- 装作不知型：暗中观察

糖宝的选择：
- 逃跑：立刻离开
- 应对：与原配对话
- 反击：反过来威胁
- 求饶：哭诉自己的处境
- 沉默：不做任何回应

可能结局：
- 被原配原谅（概率低）
- 被给钱封口（需签协议）
- 被曝光（最坏结果）
- 变成原配与糖爹的婚姻危机
```

**场景六：关系结束**

```
触发条件：
- 糖爹提出分手/糖宝主动离开
- 关系无法继续

场景描述：
[某场所 - 对话]
关系走到了尽头...

糖爹提出分手：
- "我们分开一段时间"
- 给一笔分手费
- 要求删除联系方式
- 要求保密协议

糖宝的反应选择：
- 接受：拿钱离开
- 纠缠：想要复合/要更多
- 谈判：讨论分手费
- 体面离开：不纠缠
- 报复：威胁曝光

关键决策点：
- 分手费金额谈判
- 是否签保密协议
- 未来是否还有联系
- 是否保留"朋友"关系
```

## 七、与其他模块联动

### 7.1 与直播/短视频模块联动

**联动点一：网红糖宝**

```
糖宝是网红/主播
↓ 通过直播认识糖爹
↓ 榜一大哥可能就是糖爹
↓ 直播打赏+私下包养结合
```

**联动点二：直播曝光风险**

```
糖宝直播时不小心露馅
↓ 粉丝发现异样
↓ 引发关于"被包养"的猜测
↓ 可能触发舆论危机
```

### 7.2 与演艺圈模块联动

**联动点一：娱乐圈资源置换**

```
糖宝想要进入演艺圈
↓ 糖爹承诺介绍资源
↓ 以"陪伴"换取机会
↓ 可能涉及潜规则
```

**联动点二：偶像与粉丝**

```
糖爹是某个偶像的粉丝
↓ 追星过程中的特殊关系
↓ 粉丝对偶像的"供养"
↓ 另一种形式的糖爹关系
```

### 7.3 与高端会所模块联动

**联动点一：会所是社交场合**

```
糖爹带糖宝去高端会所
↓ 遇到其他糖爹和糖宝
↓ 形成社交圈
↓ 信息交换/资源整合
```

**联动点二：会所是交易场所**

```
高端会所也是糖爹寻找糖宝的场所
↓ 妈咪/领队牵线搭桥
↓ 糖宝的另一个身份
```
