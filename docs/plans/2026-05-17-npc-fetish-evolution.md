# NPC 性癖动态变化机制 — 事件驱动的性癖演化系统

**日期：** 2026-05-17
**状态：** 已完成

---

## 背景与目标

现有性癖档案是**静态的**：由人格模板初始化后，只有通过 AI 的 `tavern_commands` 手动修改才能变化。缺少以下机制：

1. **行为感知**：NPC 经历了特定行为（露出、被强势对待、被调教等）后，性癖不会自动演化
2. **强度累积**：性癖强度是固定值，不会随经历事件增加
3. **新性癖觉醒**：NPC 经历前所未有的体验后，不会觉醒新的性癖条目
4. **隐藏偏好解锁**：`隐藏偏好` 只能通过亲密度解锁，不会因特定事件触发
5. **禁忌软化**：`绝对禁忌` 永远不变，但实际上经历后可能转变为 `可协商`

**目标：** 构建一个事件驱动的性癖动态变化系统，让 NPC 的性癖随游戏剧情自然演化。

---

## 设计原则

1. **渐进式演化**：性癖变化是缓慢的，需要多次事件累积才触发强度提升或新性癖觉醒
2. **双向变化**：性癖强度可以增加也可以减少（长期不接触某类行为，相关偏好会淡化）
3. **人格一致性**：性癖变化不能脱离人格框架，每个 NPC 的变化范围和速度由人格决定
4. **事件驱动 + 衰减机制**：事件触发变化，但长期不接触会衰减（避免所有 NPC 最终都是全性癖）
5. **AI 辅助 + 系统计算**：AI 通过 prompt 感知变化并叙事呈现，系统通过规则计算数值变化

### 三、核心演化引擎

#### 3.1 事件记录器

```typescript
// models/npcNSFWEnhancement/evolutionEngine.ts

/**
 * 记录一次性癖触发事件
 * 更新事件计数器，检查是否达到变化阈值
 */
export function 记录性癖触发事件(
  npc: NPC结构,
  事件: 性癖触发事件,
  游戏时间: string,
  事件描述: string
): { 有变化: boolean; 变化日志: 性癖变化日志[] }
```

#### 3.2 性癖变化执行器

- 已有条目 → 提升强度（上限 5）
- 在隐藏偏好中 → 升级为核心偏好
- 不存在 → 新性癖觉醒（强度 1，标记来源为 `事件觉醒`）

#### 3.3 衰减系统

```typescript
/**
 * 回合结束时调用：检查并应用性癖衰减
 * 长期未触发的性癖会逐渐淡化
 */
export function 应用性癖衰减(
  npc: NPC结构,
  当前游戏时间: string,
  当前回合数: number
): { 有变化: boolean; 变化日志: 性癖变化日志[] }
```

衰减规则：
- 默认衰减率 0.02/回合
- 5 回合内不衰减（保护期）
- 强度降为 1 的 `事件觉醒` 来源偏好移回隐藏偏好
- `人格模板` 来源且强度 ≤ 2 的核心偏好不衰减（人格基础特性）

### 五、安全保护

| 保护项 | 规则 |
|--------|------|
| 核心偏好上限 | 最多 8 条，超过后不再添加新性癖 |
| 禁忌保护 | `绝对禁忌` 不会自动移除，需 AI 显式修改 |
| 强度上限 | 所有性癖强度不超过 5 |
| 人格锚定 | 通过人格身份标签限制可觉醒的性癖范围 |
| 衰减保护 | 人格模板偏好（强度 ≤ 2）不衰减 |
