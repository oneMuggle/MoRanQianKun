# NPC NSFW 增强模块 — 表里人格 & 增强版性癖优化方案

**日期：** 2026-05-17
**状态：** 已完成

---

## 背景与目标

现有女性 NPC 的 NSFW 模块已有较完善的基础设施：
- `性癖` 和 `敏感点` 字段为扁平 `string`，缺乏结构化和分类
- `dualPersonalities`（表里人格）已有 12 个里都市人格模板，但未与性癖/敏感点联动
- `NSFW行为特征结构` 包含 `主动程度`、`反差偏好`、`特殊癖好`，但未与 NPC 个体深度绑定
- BDSM 系统、校园系统、写真系统等各自有独立数据，缺乏统一抽象

**目标：** 构建结构化的性癖分类体系、敏感点身体区域模型、表里人格增强档案，并将三者打通形成联动系统。

## 技术方案

### 一、增强版性癖分类体系（Fetish Taxonomy）

#### 1.1 性癖分类结构

```typescript
// 性癖大类
export type 性癖大类 =
  | '主导控制'     // 支配/服从/指令
  | '角色扮演'     // 职业/身份扮演
  | '感官刺激'     // 轻咬/抚摸/温度
  | '暴露窥视'     // 露出/偷窥/被拍
  | '拘束限制'     // 束缚/蒙眼/噤声
  | '语言调教'     // 羞辱/称赞/命令
  | '公共冒险'     // 半公开/险些暴露
  | '情感依赖'     // 占有/嫉妒/专属
  | '特殊偏好'     // 恋物/特定服装/特定场景

// 单个偏好条目
export interface 性癖条目 {
  类别: 性癖大类;
  子类型: string;           // 如 '轻咬耳垂', '职场角色扮演'
  强度: 1 | 2 | 3 | 4 | 5; // 1=轻微好奇, 5=强烈渴望
  解锁条件?: { 类型: '亲密度阈值' | '事件触发'; 值: number | string };
  描述: string;             // AI 描写指引
  时代适配?: string[];      // 适用的 eraId 列表，空=全时代
}

// NPC 性癖档案
export interface 性癖档案 {
  核心偏好: 性癖条目[];     // 始终生效的偏好（1-3 个）
  隐藏偏好: 性癖条目[];     // 需要解锁才暴露的偏好
  绝对禁忌: string[];       // 永远不会接受的边界
  可协商: string[];         // 可以尝试但不主动要求的
  倾向摘要: string;         // 一句话摘要，用于 prompt 快速注入
}
```

### 三、增强版表里人格档案（Enhanced Dual Personality）

#### 3.1 从字符串到结构化

当前的 `dualPersonalities` 是字符串数组，格式为 `"角色名：表——X；里——Y"`。
增强版将其升级为结构化对象：

```typescript
export interface 表里人格档案 {
  名称: string;              // 如 '职场精英', '邻家女孩'
  身份标签: string[];        // 匹配关键词：['职场', '精英', 'OL']
  
  // 表人格（外在表现）
  表: {
    性格描述: string;
    行为特征: string[];      // 外在行为模式
    对话风格: string;        // 说话方式
    服饰偏好: string;        // 穿衣风格
    社交面具: string;        // 在公众面前的形象
  };
  
  // 里人格（内在真实）
  里: {
    性格描述: string;
    行为特征: string[];      // 私下行为模式
    对话风格: string;        // 私密时的说话方式
    欲望驱动: string[];      // 内在欲望来源
    反差触发器: string;      // 什么情况下会展现里人格
  };
  
  // 联动数据（新增）
  关联偏好: 性癖条目[];      // 该人格对应的性癖推荐
  关联敏感点: 敏感点条目[];   // 该人格对应的敏感点推荐
  推荐场景: string[];        // 适合该人格的亲密场景
  激活条件: 人格激活条件;     // 何时切换为里人格
}

export interface 人格激活条件 {
  亲密度阈值?: number;
  关系状态?: string[];       // 如 ['暧昧', '恋人', '秘密关系']
  时间条件?: string;         // 如 '夜晚', '独处时'
  地点条件?: string;         // 如 '私人空间', '车内'
  事件条件?: string;         // 如 '酒精作用后', '危机事件后'
}
```

### 四、联动系统（Linkage）

#### 4.1 人格 → 性癖 → 敏感点 → 场景推荐

```typescript
// 根据 NPC 人格档案生成完整 NSFW 画像
export function 生成NSFW画像(npc: NPC结构, eraId: string): NPCNSFW画像 {
  // 1. 匹配人格模板
  const 人格 = 匹配人格模板(npc, eraId);
  
  // 2. 从人格提取关联性癖
  const 核心偏好 = 人格.关联偏好;
  const 隐藏偏好 = 根据亲密度解锁(核心偏好, npc.亲密度等级);
  
  // 3. 从人格提取关联敏感点
  const 主要敏感点 = 人格.关联敏感点;
  
  // 4. 推荐适合的场景
  const 推荐场景 = 人格.推荐场景
    .filter(s => 匹配时代(s, eraId));
  
  return { 人格, 核心偏好, 隐藏偏好, 主要敏感点, 推荐场景 };
}
```

## 设计原则

1. **向后兼容**：现有 `性癖`/`敏感点` string 字段保留，作为无结构化数据时的 fallback
2. **时代适配**：同一偏好/敏感点在不同时代使用不同措辞，由系统自动选择
3. **渐进解锁**：隐藏偏好和敏感点随亲密度推进逐步揭示，增加游戏深度
4. **摘要优先**：AI prompt 注入使用一句话摘要，避免 prompt 膨胀
5. **数据驱动**：偏好/敏感点/人格全部通过数据文件定义，便于后续扩展
