# NSFW 深化方案：NPC活人感与玩法深度增强

> 创建日期：2026-05-23
> 状态：已完成

## 一、背景与目标

### 背景
当前NSFW系统已具备完善的架构（54个人格、9大类性癖、6+子系统），但存在以下问题：
1. NPC行为模式偏被动，主要由AI驱动，缺少自主意志
2. 子系统之间联动薄弱，缺少长期目标和挑战
3. 事后护理停留于数据记录，未形成真正的情感纽带
4. 玩家缺少主动选择和策略性玩法

### 目标
1. **NPC活人感**：自主行为、情绪波动、社交网络、嫉妒系统
2. **玩法深度**：情感羁绊树、策略性玩法、跨子系统联动
3. **沉浸体验**：动态叙事、事后对话、NSFW记忆库

## 三、技术方案

### 3.1 情绪周期系统

#### 数据结构
```typescript
interface 情绪状态 {
  心情值: number;           // 0-100，影响互动响应
  心情阶段: '开心' | '平静' | '烦躁' | '低落' | '愤怒';
  情绪波动率: number;        // 0-1，人格决定的波动幅度
  最后情绪更新时间: string;
  情绪触发器?: 情绪触发记录[];
}

interface 情绪触发记录 {
  触发类型: '场景' | '对话' | '记忆' | '环境';
  触发源: string;
  情绪变化: number;
  时间: string;
}
```

#### 情绪计算逻辑
- 每个回合根据人格类型计算基础情绪波动
- 环境因素（时间、地点、天气、在场人物）产生额外影响
- 记忆触发：特定场景/对话触发过往记忆

### 3.2 NPC主动行为引擎

#### 行为类型
```typescript
type 主动行为类型 =
  | '主动邀约'      // 根据好感度/性癖/情绪发起互动
  | '拒绝反抗'      // 不符合心理防线时拒绝
  | '试探行为'      // 好感度提升中试探玩家态度
  | '情绪表达'      // 主动表达当前情绪状态
  | '社交互动'      // 与其他NPC互动
  | '嫉妒表现'      // 多NPC存在时的嫉妒行为
```

#### 触发条件
- 主动邀约：好感度 >= 40 且心情值 >= 50
- 拒绝反抗：心理防线 > 当前行为允许值
- 试探行为：好感度 30-60 之间
- 嫉妒表现：玩家与其他NPC互动后

### 3.3 社交网络系统

#### 数据结构
```typescript
interface 社交网络 {
  关系列表: NPC关系[];
  声誉值: Record<string, number>;  // 不同圈子的声誉
  流言列表: 流言[];
}

interface NPC关系 {
  目标NPC: string;
  关系类型: '友谊' | '竞争' | '敌对' | '暧昧' | '主从';
  关系强度: number;  // 0-100
  共享秘密: boolean;
}

interface 流言 {
  内容: string;
  传播范围: string[];  // 知道这个流言的NPC
  传播时间: string;
  影响: number;  // 对声誉的影响
}
```

### 3.4 嫉妒系统

#### 触发条件
- 玩家与NPC A互动后，NPC B（与A有关系）会产生嫉妒
- 嫉妒强度 = 基础嫉妒值 × 关系强度 × 人格系数

#### 嫉妒表现
- 冷淡回应
- 主动质问
- 暗中报复
- 自我伤害（极端人格）

### 3.5 情感羁绊树

#### 数据结构
```typescript
interface 情感羁绊树 {
  里程碑列表: 情感里程碑[];
  当前分支: 剧情分支;
  结局倾向: '恋人' | '炮友' | '主从' | '陌路' | '知己';
  羁绊值: number;  // 0-100
}

interface 情感里程碑 {
  名称: string;
  描述: string;
  达成时间: string;
  影响: {
    羁绊值变化: number;
    结局倾向变化: Record<string, number>;
    解锁内容?: string[];
  };
}
```

#### 里程碑示例
- 第一次牵手
- 第一次约会
- 第一次NSFW
- 第一次事后拥抱
- 第一次吃醋
- 第一次主动邀约
- 第一次说"我爱你"

### 3.6 策略性玩法

#### 氛围系统
```typescript
interface 氛围状态 {
  地点系数: number;      // 地点适合度
  时间系数: number;      // 时间适合度
  前戏系数: number;      // 前戏充分度
  道具系数: number;      // 道具匹配度
  情绪系数: number;      // NPC情绪适合度
  综合评分: number;      // 0-100
}
```

#### 节奏控制
- 过快推进：降低成功率，可能触发拒绝
- 适当节奏：提升羁绊值，解锁更多内容
- 节奏判断：根据人格和当前状态动态调整

### 3.7 跨子系统联动

#### 事件总线
```typescript
interface 跨系统事件 {
  源系统: string;
  事件类型: string;
  影响系统: string[];
  影响数据: Record<string, any>;
}
```

#### 联动示例
- 校园露出 → 声誉下降 → 影响其他NPC态度
- 网约车事件 → 产生记忆 → 影响后续互动
- 写真泄露 → 社交网络传播 → 影响多人关系

### 3.8 统一后果系统

#### 后果类型
- 短期后果：情绪变化、态度变化
- 中期后果：关系变化、声誉变化
- 长期后果：人格演化、结局倾向

#### 后果传播
- 直接后果：影响当事NPC
- 间接后果：通过社交网络影响其他NPC
- 系统后果：影响游戏状态和可用内容

## 六、风险评估与依赖

### 风险
| 风险 | 影响 | 应对 |
|------|------|------|
| AI提示词过长 | 上下文超限 | 优化提示词结构，使用动态注入 |

| 平衡性问题 | 玩法失衡 | 充分测试，收集反馈 |
