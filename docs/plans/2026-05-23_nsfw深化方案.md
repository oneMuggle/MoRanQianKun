### 1.2 存在的问题

**问题 A：系统孤岛 -- 已实现的模型未接入游戏主循环**

这是当前最严重的问题。以下关键系统已完整实现代码，但在 `useGame.ts` 和 `sendWorkflow/` 中**没有任何调用**：

1. `npcAutonomy.ts` -- NPC 主动行为引擎（240行），计算结果从未在回合流程中使用
2. `crossSystemLinker.ts` -- 跨子系统联动（272行），事件总线从未被触发
3. `dynamicNarrative.ts` -- 动态叙事约束（169行），提示词未注入
4. `aftercareDialogue.ts` -- 事后对话引导（160行），提示词未注入
5. `jealousySystem.ts` -- 嫉妒系统仅被 npcAutonomy 引用，未在主流程中计算

**问题 B：NPC "活人感" 缺失的具体表现**

| 维度 | 当前状态 | 期望状态 |
|------|----------|----------|
| 自主行动 | npcAutonomy 有代码但未接入 | 每回合根据欲望/情感/关系自主发起互动 |
| NPC间互动 | socialNetwork 有数据结构但无行为引擎 | NPC 之间有 gossip、互助、冲突 |
| 记忆连贯性 | nsfwMemory 存储事件但检索简单 | 基于情境的 Episodic Memory（情景记忆） |
| 情感深度 | 心情值单一维度 (0-100) | 复合情感状态机（爱慕+嫉妒+不安等组合） |
| 日常模式 | 无 | NPC 有自己的作息、习惯、偏好模式 |
| 内在动机 | 无显式欲望/目标系统 | NPC 有独立于玩家的欲望和短期目标 |

**问题 C：NSFW 玩法缺少的机制**

| 机制 | 当前状态 | 期望状态 |
|------|----------|----------|
| 前戏/节奏控制 | strategySystem 有评分但未接入 | 分阶段互动选项，节奏影响成功率与叙事质量 |
| 偏好/禁忌 | fetishTaxonomy 有分类但无动态底线 | 每个 NPC 有独特喜好和不可逾越的底线 |
| 事后关系演化 | aftercareSystem 有数据但无长期影响 | 护理质量持久改变 NPC 性格和行为模式 |
| 环境/场景影响 | sceneModifiers 有计算但未充分使用 | 地点、时间、风险实时影响互动选项和叙事 |
| 连锁反应 | butterflyEffects 有概念但未实现 | 一次 NSFW 事件影响后续 NPC 行为、社交关系、剧情走向 |
| 潜能/觉醒 | fetishPotential 有数据但觉醒条件僵化 | 随亲密度和事件积累渐进解锁新互动 |

**问题 D：Prompt Token 成本控制缺失**

当前 `npcNSFWEnhancement.ts` 生成的注入文本已经很长（~270行代码生成），加上 personalityProfiles（64KB 的人格档案库）和各个子系统的注入，单个 NPC 的 prompt 注入可能达到数百 token。多 NPC 在场时会迅速膨胀。缺少：
- 注入内容的优先级分级（核心/重要/可选）
- Token 预算控制机制
- 动态裁剪策略

## 二、设计方案

### 2.1 NPC 活人感系统

#### 2.1.1 自主决策引擎升级（核心改造）

**现有问题**：`npcAutonomy.ts` 有判定函数但概率固定、缺少内在动机。

**设计方案**：引入 NPC 内在动机系统。

```
新增模型文件: models/npcNSFWEnhancement/npcMotivation.ts
```

**核心概念**：
- `内在欲望`：NPC 独立于玩家的基本欲望（被认可、刺激感、安全感、归属感、支配感等）
- `短期目标`：基于当前情感状态和欲望优先级生成的短期意图
- `满足度`：各欲望的当前满足程度，未满足的欲望会推动自主行为

```typescript
export type 欲望维度 =
  | '被认可'       // 渴望被关注和赞赏
  | '刺激感'       // 渴望新鲜体验和冒险
  | '安全感'       // 渴望稳定和可预测
  | '归属感'       // 渴望亲密关系和陪伴
  | '支配感'       // 渴望掌控和主导
  | '自由感';      // 渴望无拘束和自我表达

export interface 欲望状态 {
  维度: 欲望维度;
  当前满足度: number;    // 0-100
  衰减速率: number;      // 每回合衰减量
  优先级系数: number;    // 人格决定的权重
  最近满足时间: string;
}

export interface NPC内在动机 {
  欲望状态: Record<欲望维度, 欲望状态>;
  当前主导欲望: 欲望维度;
  短期目标: 短期目标条目[];
  最后更新时间: string;
}

export interface 短期目标条目 {
  目标描述: string;        // "今晚想和他独处"、"需要发泄压力"
  驱动欲望: 欲望维度;
  优先级: number;          // 0-100
  时间窗口: string;        // 目标有效的时间范围
  行为倾向: 主动行为类型[]; // 可能采取的为类型
}
```

**NPC 自主行为生成流程**：

```
回合开始
  |
  v
[欲望衰减] 所有欲望满足度自然衰减
  |
  v
[情感输入] 读取当前情绪、羁绊值、嫉妒状态
  |
  v
[目标生成] 基于未满足欲望 + 情感状态生成短期目标
  |
  v
[行为候选] 从短期目标推导可能的行为候选列表
  |
  v
[概率判定] 根据好感度、亲密度、心理防线、地点过滤
  |
  v
[选择输出] 按优先级排序，选择 0-2 个行为作为本回合 NPC 自主行动
  |
  v
[注入上下文] 将自主行为作为 AI 提示词的一部分，引导 AI 生成 NPC 主动行为的叙事
```

#### 2.1.2 深层记忆系统（Episodic Memory）

**现有问题**：`nsfwMemory.ts` 的记忆是基于关键词匹配的简单检索，缺少情境关联。

**设计方案**：在现有 NSFW 记忆基础上，增加情景记忆层。

```
新增模型文件: models/npcNSFWEnhancement/episodicMemory.ts
```

**核心概念**：
- `情景记忆`：带有情境标签（时间、地点、在场人物、情感状态）的记忆片段
- `记忆锚点`：关键事件（初夜、人格翻转、嫉妒爆发）作为长期记忆的锚
- `情境检索`：当前情境（时间、地点、情绪）与记忆情境匹配时，自动触发相关回忆

```typescript
export interface 情景记忆条目 {
  id: string;
  事件摘要: string;           // "深夜客栈，第一次亲密"
  发生时间: string;
  地点: string;
  在场人物: string[];
  当时情感: 心情阶段;
  情感强度: number;            // 事件时的情感强度
  重要度: number;              // 1-10
  情境标签: string[];          // ['深夜', '私密', '首次', '温柔']
  关联NPC: string[];
  记忆强度: number;            // 0-100, 随时间和回忆次数变化
  衰减系数: number;            // 默认 0.98
  回忆次数: number;            // 被回忆的次数（强化记忆）
  最后回忆时间?: string;
  // --- 新增：记忆烙印 ---
  烙印效果?: 记忆烙印效果;     // 对性格/行为的持久影响
}

export interface 记忆烙印效果 {
  类型: '偏好形成' | '创伤反应' | '情感锚定' | '行为模式';
  描述: string;
  触发条件: string[];          // 什么情境下会被激活
  行为影响: string;            // 对 NPC 行为的影响描述
}
```

**性能控制**：
- 单个 NPC 最多保留 30 条情景记忆
- 重要度 < 3 且记忆强度 < 10 的记忆自动清除
- 注入 prompt 时只取 Top 3（按记忆强度 * 情境匹配度排序）

#### 2.1.3 复合情感状态机

**现有问题**：`emotionSystem.ts` 的心情值是单一维度的 0-100，无法表达复杂情感。

**设计方案**：在现有心情值基础上，增加复合情感层。

```
新增模型文件: models/npcNSFWEnhancement/compoundEmotion.ts
```

**核心概念**：
- `基础心情`：保留现有的 0-100 心情值（已实现）
- `情感维度`：在基础心情之上叠加 5 个独立情感维度
- `主导情感`：当前最强的情感维度，用于提示词注入
- `情感冲突`：当两个对立情感同时高强度时，产生行为矛盾

```typescript
export type 情感维度 =
  | '爱慕'       // 对玩家的好感/依恋
  | '嫉妒'       // 对其他 NPC 的嫉妒
  | '不安'       // 对关系/未来的担忧
  | '兴奋'       // 对刺激的渴望
  | '内疚';      // 对行为的道德反思

export interface 复合情感状态 {
  基础心情: number;             // 0-100 (沿用现有)
  情感维度: Record<情感维度, number>; // 各维度 0-100
  主导情感: 情感维度;
  情感冲突: 情感冲突条目[];
  最后更新时间: string;
}

export interface 情感冲突条目 {
  维度A: 情感维度;
  维度B: 情感维度;
  冲突强度: number;            // 0-100
  行为表现: string;            // "想靠近却又退缩"、"依恋但生气"
}
```

**情感影响行为规则**：
- 爱慕高 + 不安高 = 黏人但试探性（"你是不是喜欢别人？"）
- 嫉妒高 + 内疚低 = 直接质问/冷淡
- 嫉妒高 + 内疚高 = 自我压抑、暗自难过
- 兴奋高 + 内疚高 = 矛盾行为（主动靠近后突然退缩）

#### 2.1.4 NPC 间 gossip 引擎

**现有问题**：`socialNetwork.ts` 有流言数据结构，但没有 gossip 行为的自动生成和消费。

**设计方案**： gossip 引擎自动根据 NSFW 事件、社交关系生成流言，并影响 NPC 态度。

```
新增模型文件: models/npcNSFWEnhancement/gossipEngine.ts
```

**核心流程**：

```
[NSFW事件发生]
  |
  v
[流言生成] 根据事件类型、地点公开程度、在场 NPC 生成流言
  |
  v
[传播判定] 流言通过社交网络传播（信任度高的 NPC 间传播更快）
  |
  v
[态度影响] 知道流言的 NPC 对目标 NPC 的态度自动调整
  |
  v
[主动行为] 可能被 gossip 刺激的 NPC 发起"质问"或"疏远"行为
```

  | '深度前戏'     // 更亲密的身体互动

## 四、风险评估

### 4.1 技术风险

| 风险 | 等级 | 影响 | 应对措施 |
|------|------|------|----------|
| Prompt Token 超限 | 高 | 多 NPC 在场时上下文爆炸 | 阶段二即实现智能裁剪；单 NPC 预算上限 500 tokens |
| 状态膨胀 | 中 | IndexedDB 存储压力 | 情景记忆上限 30 条/人；旧记忆自动清除；日志类数据限制 N 条 |
| 性能下降 | 中 | 回合处理延迟 | 自主行为只在在场 NPC 上计算； gossip 传播每回合限制 N 步 |
| 集成复杂度 | 高 | responseCommandProcessor 已有 731 行，继续增加 | 拆分 XML 解析器为独立模块；每个标签解析独立函数 |
| 状态一致性 | 中 | 多系统更新可能导致不一致 | 使用不可变更新模式；回合处理采用事务式批量更新 |

### 4.2 设计风险

| 风险 | 等级 | 影响 | 应对措施 |
|------|------|------|----------|
| AI 叙事质量不稳定 | 中 | 自主行为/gossip 的 AI 表达可能不符合预期 | 精心设计的提示词模板；分阶段 A/B 测试 |
| 玩法过于复杂 | 低 | 玩家认知负担增加 | 所有新系统默认开启但 UI 可选择性展示 |
| NPC 行为过于主动 | 低 | 玩家失去控制感 | 自主行为概率受好感度/亲密度严格控制 |

### 4.3 兼容性

- 所有新增字段使用 optional (`?`) 标记，不破坏现有存档
- 新增模型文件完全独立，不修改已有文件的导出接口
- `responseCommandProcessor.ts` 的 XML 解析采用"无标签则跳过"策略，向后兼容
