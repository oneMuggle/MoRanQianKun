## 一、目标与背景

在现有 BDSM 论坛 + 寻主召奴系统的基础上，构建完整的 **BDSM 关系管线（Relationship Pipeline）**，将碎片化的互动整合为有深度、有递进、有反馈的玩法系统。

### 核心设计原则

1. **见面放在主剧情中**：正式见面、约会、调教场景由主剧情 AI 生成，纳入正文叙事
2. **手机做前置沟通**：论坛浏览、帖子联系、私聊协商放在手机私聊应用（CampusChatApp）
3. **循序渐进**：关系从初识到固化，每个阶段有明确的解锁条件和玩法内容
4. **任务驱动**：通过调教任务系统推动关系发展，任务生成由 AI 根据上下文动态产生
5. **契约约束**：口头约定 → 书面契约 → 信物交换，层层递进的关系绑定
6. **安全优先**：安全词、退出机制、底线保护贯穿始终

## 七、关系阶段详解

### 7.1 阶段推进表

| 阶段 | 服从度 | 完成任务 | 完美服从 | 违约 | 契约要求 | 解锁玩法 |
|------|--------|----------|----------|------|----------|----------|
| **初识** | 0-19 | 0+ | 0 | 0 | 无 | 论坛联系、私聊沟通 |
| **试探** | 20-39 | 2+ | 0+ | 0 | 口头约定 | 入门任务、日常指令（称呼/行为） |
| **确立** | 40-59 | 5+ | 1+ | 0 | 书面契约 | 中级任务、忠诚考验、见面场景 |
| **深入** | 60-79 | 10+ | 3+ | ≤1 | 书面契约+ | 高级任务、公开挑战、技能训练 |
| **固化** | 80-100 | 20+ | 8+ | 0 | 信物交换 | 极限任务、信物任务、专属场景 |

### 7.2 各阶段 AI 叙事指引

| 阶段 | 叙事风格 | 任务特征 |
|------|----------|----------|
| 初识 | 神秘、谨慎、试探性 | 轻度服从测试，心理建设 |
| 试探 | 逐渐信任，偶尔犹豫 | 日常指令为主，轻度技能训练 |
| 确立 | 信任建立，主动配合 | 忠诚考验，中级训练，见面互动 |
| 深入 | 深度信任，享受过程 | 公开挑战，高级训练，复杂任务链 |
| 固化 | 完全投入，身份认同 | 极限挑战，信物交换，专属故事线 |

## 八、任务系统详解

### 8.1 任务类型特征

| 类型 | 描述 | 示例 | 服从度变化 |
|------|------|------|-----------|
| **服从测试** | 测试 NPC 对指令的执行意愿和能力 | "今天一整天用特定称呼叫我" | +5~15 |
| **忠诚考验** | 考验 NPC 对关系的忠诚度和排他性 | "今天不和其他人单独相处" | +10~20 |
| **技能训练** | 训练特定技巧或服务能力 | "学习泡一种特别的茶" | +8~18 |
| **心理建设** | 心理层面的引导和强化 | "写下三个你欣赏自己的理由" | +5~10 |
| **公开挑战** | 在公共场合的暧昧任务（高风险） | "在走廊上对我微笑并鞠躬" | +15~30 |
| **日常指令** | 每日例行轻量任务 | "每天早安问候" | +3~8 |
| **信物任务** | 与信物交换/保管相关的任务 | "保管好这个信物直到下次见面" | +10~25 |
| **契约履行** | 契约条款的具体执行 | "完成契约第3条约定的事项" | +8~20 |

### 8.2 任务生命周期

```
AI 生成任务 → 玩家/NPC 接受 → 状态变为「进行中」
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
              主剧情体现          玩家报告完成          超时/放弃
                    │                   │                   │
                    ▼                   ▼                   ▼
              AI 评价结果        AI 评价 + 奖惩        状态变为「已失败」
                    │                   │
                    ▼                   ▼
              服从度变化          服从度变化
              里程碑记录          里程碑记录
                    │                   │
                    ▼                   ▼
              关系阶段判定        关系阶段判定
```

## 九、契约系统详解

### 9.1 契约类型

| 类型 | 条款数 | 约束力 | 解锁阶段 |
|------|--------|--------|----------|
| **口头约定** | 2-3 条 | 弱，可随时修改 | 试探 |
| **书面契约** | 4-6 条 | 中，违约记录 | 确立 |
| **信物交换** | 3-5 条 | 强，情感绑定 | 固化 |

### 9.2 契约生命周期

```
协商条款 → AI 生成条款建议 → 双方确认 → 缔结
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
              正常履行            触发违约            主动解除
                    │                   │
                    ▼                   ▼
              定期评估            记录违约次数
              续约/升级            超过阈值 → 关系降级
```

## 十三、风险评估

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| AI 生成的任务质量不稳定 | 高 | 严格的 JSON schema 约束 + 回退到预设任务池 |
| 主剧情 prompt 过长 | 中 | 仅注入活跃任务（最多 5 个）+ 日常指令摘要 |
| 关系推进过快/过慢 | 中 | 设置明确的数值阈值 + AI 判定辅助 |
| 任务系统与 SM 系统数据冲突 | 中 | 共享同一数据源，管线作为操作层 |
| 手机端 UI 复杂度爆炸 | 中 | 面板式设计，一次只显示一个面板 |
| 非校园纪元误出现 | 低 | 仅在校园纪元 + BDSM 论坛启用时生效 |
| 存档兼容性问题 | 低 | 新字段有默认值，向后兼容 |
