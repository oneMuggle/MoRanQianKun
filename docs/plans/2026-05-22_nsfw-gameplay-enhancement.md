# NSFW 玩法增强方案 — 可玩性/深度/多样性三期路线图

> 创建日期：2026-05-22
> 状态：实施中 — Phase 1 ✅ | Phase 2 ✅ | Phase 3 ✅ | Phase 4 ✅

## 一、现状诊断

### 1.1 已有资产盘点

当前 NSFW 系统包含 **9 个子系统**，数据模型极为丰富（100+ 类型定义），但**玩法深度严重不足**：

| 子系统 | 模型深度 | 玩家可玩性 | 反馈循环 | 游戏循环 |
|---|---|---|---|---|
| NPC 增强（28 文件） | ★★★★★ | ★☆☆☆☆ | ★★☆☆☆ | 无 |
| Campus（6+ 内部系统） | ★★★★☆ | ★★★☆☆ | ★★★☆☆ | 部分完整 |
| BDSM | ★★★★☆ | ★★★☆☆ | ★★★★☆ | 部分完整 |
| 桌游（7 种游戏） | ★★★★☆ | ★★★★☆ | ★★★★☆ | 完整 |
| 暴露风险 | ★★★★☆ | ★★☆☆☆ | ★★☆☆☆ | 缺失 |
| 写真约拍 | ★★★★☆ | ★★★☆☆ | ★★★☆☆ | 部分完整 |
| 网约车 | ★★★☆☆ | ★★☆☆☆ | ★★☆☆☆ | 缺失 |
| 酒吧 | ★★★☆☆ | ★★☆☆☆ | ★★☆☆☆ | 缺失 |
| Outdoor | ★★☆☆☆ | ☆☆☆☆☆ | ☆☆☆☆☆ | 无（未实现） |

### 1.2 核心问题

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **被动体验** | NPC 增强（性癖、敏感点、人格演化）最丰富的数据模型，零玩家交互，全靠 AI 叙事被动触发 | 玩家只是"看故事"，不是"玩游戏" |
| **缺主动玩法** | 暴露风险有 12+ 因子计算张力，但玩家无"潜行/躲避"操作；写真有风险计算但无"拍摄管理"操作 | 所有子系统都变成概率掷骰，缺乏技巧/策略 |
| **无资源经济** | 没有金钱/精力/时间/声誉的统筹经济层，子系统各自为政 | 缺少跨子系统的资源博弈和取舍 |
| **无主动后果解决** | 后果（谣言、声誉下降、法律风险）被动积累，玩家无"危机公关"玩法 | 后果变成数值惩罚，不是 gameplay |
| **多样性偏科** | Campus 有 6+ 内部系统，Outdoor 零实现，酒吧只有 1 个循环 | 场景严重偏科，缺乏时代/文化差异 |
| **无进度/元游戏** | 没有持久解锁、成就树、多周目继承 | 玩久了缺乏目标感 |

### 1.3 设计目标

1. **可玩性**：每个子系统至少有一个**主动操作循环**（不只是概率掷骰）
2. **玩法深度**：引入**资源管理**、**策略取舍**、**技巧曲线**
3. **多样性**：补齐 Outdoor，增加跨场景协同，引入**多人场景**和**文化变体**

## 二、三期路线图

### 第三期：玩法核心增强（~20h）

**主题**：为现有子系统注入主动玩法循环

#### Phase 1: 性癖探索引擎（~6h）

**目标**：把 NPC 增强从被动追踪变成**可主动探索**的玩法。

##### 1.1 性癖发现小游戏
- [x] **文件**: `models/npcNSFWEnhancement/discovery/fetishDiscovery.ts` ✅
- **内容**:
  - `试探交互()`：玩家选择试探类型（言语试探/肢体试探/场景试探），基于亲密度和信任度计算成功率
  - `深度探索()`：发现隐藏性癖后的深入探索，有"安全/激进"两档策略
  - `性癖解锁树()`：已解锁性癖作为前置，解锁更深层次性癖（形成解锁树）
  - `探索风险评估()`：激进探索可能导致信任度下降或关系倒退
- **玩法**：玩家主动选择探索路径，不是等 AI 叙事被动触发
- **依赖**: `fetishTaxonomy.ts`, `preferenceDrift.ts`
- **风险**: 低

##### 1.2 敏感点主动探索
- [x] **文件**: `models/npcNSFWEnhancement/discovery/sensitivePointDiscovery.ts` ✅
- **内容**:
  - `探索敏感点(部位, 方式)`: 玩家选择探索哪个部位、用什么方式
  - `敏感度反馈循环`: NPC 反应（语音/表情/身体语言）作为正/负反馈
  - `解锁敏感点技巧`: 发现"XX 部位对 YY 方式特别敏感"等技巧
  - `技巧组合系统`: 发现的技巧可以组合使用，产生协同效果
- **玩法**：类似"技能树探索"，玩家逐步解锁 NPC 的敏感点地图
- **依赖**: `sensitiveZones.ts`, `sensitivePointEvolution.ts`
- **风险**: 低

##### 1.3 人格演化主动触发
- [x] **文件**: `models/npcNSFWEnhancement/discovery/personalityTrigger.ts` ✅
- **内容**:
  - `触发表里人格切换()`: 玩家通过特定行为序列触发人格切换（不是随机）
  - `欲望觉醒条件`: 满足特定条件（亲密次数、性癖解锁数、信任度阈值）触发欲望觉醒
  - `创伤关闭/解放突破`: 负面/正面事件链导致的心理状态跃迁
  - `人格演化路径图`: 可视化展示 NPC 从初始状态到当前状态的演化路径
- **玩法**：玩家有意识地引导 NPC 人格演化，不是被动等待
- **依赖**: `personalityProfiles.ts`, `personalityEvolution.ts`
- **风险**: 中 — 需要与现有演化引擎深度集成

##### 1.4 探索 UI 组件
- [x] **文件**: `components/features/NSFW/FetishExplorer.tsx` ✅
- **内容**:
  - 性癖发现进度树（已解锁/未解锁/前置条件）
  - 敏感点探索地图（身体 SVG + 已探索区域高亮）
  - 人格演化时间线
  - 当前探索策略选择（安全/激进/暂停）
- **依赖**: 步骤 1.1, 1.2, 1.3
- **风险**: 低

##### 1.5 集成到 Prompt
- [x] **文件**: `prompts/runtime/npcNSFWEnhancement.ts` ✅
- **内容**: 注入当前探索进度、已解锁性癖、活跃敏感点技巧、人格演化阶段
- **依赖**: 步骤 1.1-1.3
- **风险**: 低

#### Phase 2: 暴露风险主动玩法（~5h）

**目标**：把暴露风险从概率掷骰变成**策略选择 + 技巧操作**。

##### 2.1 潜行/规避机制
- **文件**: `models/exposureNSFW/stealthMechanics.ts`
- **内容**:
  - `选择潜行策略()`: 低调行动/快速通过/伪装/分散注意力
  - `潜行成功判定()`: 基于策略选择 + 服装暴露度 + 地点隐私 + 时间 + NPC 数量
  - `反侦察技巧()`: 解锁后可降低被发现的概率（类似技能升级）
  - `紧急应对()`: 被发现后的即时选择（逃跑/解释/威胁/贿赂），不同选择影响后果严重度
- **玩法**：不再是纯概率，玩家有策略选择和即时应对
- **依赖**: 现有 `types.ts`
- **风险**: 低

##### 2.2 谣言主动管理
- **文件**: `models/exposureNSFW/rumorManagement.ts`
- **内容**:
  - `反制谣言()`: 辟谣/转移注意力/反向造谣/收买关键人物
  - `谣言溯源()`: 追踪谣言源头，选择是否正面解决
  - `声誉修复()`: 通过特定行为（慈善/公开露面/关系修复）恢复声誉
  - `谣言利用()`: 某些情况下可以利用谣言（如故意制造暧昧氛围）
- **玩法**：危机公关小游戏，有策略深度
- **依赖**: 现有 `rumorSystem.ts`
- **风险**: 低

##### 2.3 场景互动玩法
- **文件**: `models/exposureNSFW/sceneInteraction.ts`
- **内容**:
  - `场景互动选项()`: 每个场景有独特的互动选项（如图书馆有书架可躲、公园有小路可走）
  - `环境利用()`: 利用场景特性降低风险（如雨天减少路人、深夜降低可见度）
  - `道具系统()`: 携带道具（雨伞/外套/面具）改变暴露风险
- **玩法**：场景不再是数值背景板，而是可互动的游戏元素
- **依赖**: 现有 `sceneTemplates.ts`
- **风险**: 中

##### 2.4 更新 Exposure UI
- **文件**: `components/features/NSFW/ExposurePanel.tsx`（新建）
- **内容**:
  - 当前潜行策略选择按钮

### 第四期：跨系统协同与经济层（~18h）

**主题**：引入资源管理和跨子系统协同

#### Phase 5: NSFW 资源经济层（~6h）

**目标**：建立统一的 NSFW 资源经济系统，连接各子系统。

##### 5.1 NSFW 资源类型
- **文件**: `models/nsfwCore/resources.ts`

  - `亲密度代币()`: 亲密互动获得的代币，可解锁新玩法/道具
  - `风险预算()`: 每回合有有限的"风险预算"，高风险行为消耗更多

  - `技能升级()`: 消耗代币升级探索技巧/潜行能力/拍摄技巧
  - `限时特惠()`: 随机出现的特殊道具（节日/特殊事件）

## 六、实施建议

### 优先级排序

1. **最高优先**：Phase 1（性癖探索） — 影响最核心的 NPC 互动体验
2. **高优先**：Phase 2（暴露主动玩法） — 补齐最大玩法缺口
3. **高优先**：Phase 3（写真管理） — 现有子系统玩法化
4. **中优先**：Phase 5（资源经济） — 跨子系统连接层
5. **中优先**：Phase 4（Outdoor） — 补齐缺失子系统
6. **低优先**：Phase 6-10 — 锦上添花

### 每阶段验收标准

| Phase | 验收标准 |
|-------|----------|
| Phase 1 | 玩家能主动探索性癖、敏感点、人格演化，UI 有可视化进度 |
| Phase 2 | 暴露不再是纯概率，有潜行策略选择和即时应对 |
| Phase 3 | 写真有拍摄决策、后期处理、泄露应急玩的主动操作 |
| Phase 4 | Outdoor 可玩，有运动场景互动和后果系统 |
| Phase 5 | 有统一的资源管理面板，各子系统消耗/产出资源 |
| Phase 6 | 跨子系统事件链可触发，多角色关系可管理 |
| Phase 7 | 后果有应对策略选择，法律/道德系统可操作 |
| Phase 8 | 多人写真、多人暴露、派对场景可玩 |
| Phase 9 | 成就树完整，周目继承和挑战模式可用 |
| Phase 10 | 至少 2 种文化变体可切换 |

## 七、风险评估与缓解

| 风险 | 等级 | 影响 | 缓解策略 |
|------|------|------|----------|
| 玩法设计与 AI 叙事不兼容 | 高 | AI 可能忽略玩家选择 | 在 prompt 中强化玩家选择的叙事指令 |
| 资源经济数值不平衡 | 中 | 某些行为过于便宜或昂贵 | 提供数值配置接口，通过测试调优 |
| 多人场景复杂度爆炸 | 中 | NPC 数量增加导致状态管理困难 | 限制同时参与的 NPC 数量（2-3） |
| 文化变体 prompt 质量 | 中 | 不同文化的 AI 叙事质量不一致 | 每种文化独立测试 prompt |
| 向后兼容问题 | 高 | 现有存档数据损坏 | 所有新增字段使用可选类型，提供数据迁移 |
