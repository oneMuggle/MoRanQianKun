# 2026-05-05_开局环境剧情预设.md

## 背景与目标

开局向导的第四步「开局配置」当前只有切入偏好、关系侧重、同人融合三个通用配置项，缺少对具体时代的环境剧情预设支持。各个子纪元（如校园纪元下的 `contemporary_campus`）在 era 定义中已经有丰富的 `openingScenes`、`characterArchetypes`、`writingSamples`、`liMode.sceneTypes` 等数据，但这些内容没有在新建游戏流程中被使用。

**目标：** 在开局配置步骤中展示当前所选子纪元的环境剧情预设，让用户在开局前就能选择具体的场景、角色原型等，作为 AI 开局的引导素材。先从**校园纪元**（`contemporary_campus`）开始实施，后续可复用到其他纪元。

## 技术方案

### 数据流

```
选择时代(Step 1) → 查找对应 EraNode → 提取 openingScenes/archetypes/writingSamples
       ↓
Step 4 开局配置 → 展示可选项（场景卡片、角色原型、写作示例）
       ↓
用户勾选 → 存入 openingConfig 新字段
       ↓
开局故事生成 → 注入 system prompt 引导 AI
```

### 新增 OpeningConfig 字段

在 `types/index.ts` 的 `OpeningConfig` 中扩展：

```typescript
interface OpeningConfig {
    // ... 现有字段 ...
    /** 选中的开局场景 ID */
    selectedSceneId?: string;
    /** 选中的角色原型 ID 列表 */
    selectedArchetypeIds?: string[];
    /** 选中的写作示例 ID 列表 */
    selectedWritingSampleIds?: string[];
}
```

### Step 4 UI 新增区块

在「关系侧重」和「同人融合」之间插入新区块「环境剧情预设」：

1. **场景选择**（单选卡片）：展示当前子纪元的 `openingScenes`，每个场景显示 name + description
2. **角色原型**（多选标签）：展示当前子纪元的 `characterArchetypes`，显示 name + description
3. **写作风格**（多选）：展示 `writingSamples`，显示 title + excerpt（截断）
4. **里模式场景类型**（仅当里模式开启时显示）：从 `liMode.sceneTypes` 中选取

### 校园纪元特有内容

校园纪元 (`contemporary_campus`) 已有：
- **6 个开局场景**：图书馆自习、社团招新、毕业典礼、深夜实验室、操场夜跑、食堂偶遇
- **6 个角色原型**：学霸、社团达人、隐形大佬、叛逆者、温柔学长、神秘转学生
- **2 个写作示例**：期末图书馆、社团招新日
- **6 种里模式场景类型**：图书馆自习室、社团活动室、天台约会、实验室独处、操场夜跑、毕业晚会后

这些将作为首批展示内容。

### 时代预设与开局配置的桥接

现有的 `开局预设方案列表` 每个方案有 `worldConfig.时代配置ID`，可以据此自动匹配到对应子纪元的 `openingScenes` 等数据。当用户选择某个预设方案后，自动加载该时代的环境剧情预设选项。

## 风险评估与依赖

| 风险 | 等级 | 应对 |
|------|------|------|
| 其他时代暂无丰富的 openingScenes 数据 | LOW | 无数据时优雅降级，显示"暂无预设" |
| `OpeningConfig` 类型变更影响现有存档加载 | MEDIUM | 新增字段全部 optional，`规范化开局配置` 提供默认值 |
| prompt 过长导致 token 超限 | LOW | 写作示例只取 excerpt 前 100 字，场景只注入 name+description |
