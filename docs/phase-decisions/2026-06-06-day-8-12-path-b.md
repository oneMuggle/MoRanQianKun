## 决策背景

Day 8-12 spec 假设：开启 `noUncheckedIndexedAccess` 后 L1（`models/**/*.ts` glob）会暴露 80+ 错误，5 天可修复。

**实测**：暴露 **411 个错误**（远超 spec 预期，也远超 spec BLOCKED 阈值 150）。

## 三轮迭代

### 第 1 轮（Path A 初版）
6 个非核心子目录加 exclude，但 `models/**/*.ts` glob 仍命中其他大量非核心子目录。L1 错误数 **245**。

### 第 2 轮（Path A 完整版）
扩展到 8 个 exclude，仍 245 错误。**根因**：TypeScript `exclude` 不能阻止 cross-imports 触发的类型检查。

### 第 3 轮（Path B）
放弃 glob，改 **26 文件显式 include + 23 子目录 exclude**。L1 错误数 = **0**。

## spec 偏差（重要）

Path B 实际 L1 范围**比 spec 收窄**：

| 类别 | spec 要求 | 实际 L1 | 偏差原因 |
|---|---|---|---|
| 模型核心文件 | 17 个 | 26 个（多了 9 个） | 新增了 world/story/relationship/eventTrigger/narrativeGrammar/api-config/appRegistry/theme-visual/storyPlan/heroinePlan/novelDecomposition/novelWriting/installedApps/nsfwApps |
| 模型核心文件（spec 列但未入 L1） | — | 8 个被推迟 | social/intimacy/campusPhone/index/system/eraDevice/era-config/game-settings 因 cross-import 拉入 L2/L3 子目录 |
| 服务层 | dbService.ts + services/ai/** | **0 个** | 全部推迟到 L2/L3 |
| 非核心子目录排除 | spec 列 6 个 | 23 个 | 包括 avg/bdsmNSFW/boardGameNSFW/campusNSFW/dailyTown/era-device/exploration/exposureNSFW/outdoorNSFW/property/urbanDriverNSFW/domain/game/planning/fandomPlanning 等 |

## 为什么 L1 不含 services？

`dbService.ts` 通过 `services/dbService/types.ts` 拉入 `models/system.ts` 触发 82 错误；`services/ai/**` 通过 `utils/apiConfig.ts` + `prompts/runtime/*` 拉入 200+ 错误。如果硬塞这两个 service 进 L1，错误数会从 6 个 nullish guard 扩大到 200+ 处修改。

## 推迟到 L2/L3 的处理计划

- Day 19-25 阶段：开启 L2 strict（含 social/intimacy/system 等）
- Day 26-30 阶段：补漏 + 启 L3 strict
- services/dbService.ts 在 L2 推进时纳入
- services/ai/** 在 L2 推进时纳入
