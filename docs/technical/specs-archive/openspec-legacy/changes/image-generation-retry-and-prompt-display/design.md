# 文生图重试机制与提示词显示 - 技术设计

## Context

### 当前状态

当前项目的文生图功能分为两个阶段：
1. **提示词生成阶段**：调用词组转化器将 NPC/场景数据转换为生图提示词
2. **图片生成阶段**：调用图片生成服务（如 NovelAI、SD WebUI、ComfyUI 等）生成图片

当前实现（见 `hooks/useGame/npcImageWorkflow.ts` 和 `sceneImageWorkflow.ts`）：
- 提示词生成和图片生成在同一个 try-catch 块中执行
- 失败时直接抛出异常，提示词信息虽然保存在任务记录中，但没有明确的「保留提示词供用户查看」机制
- 没有重试逻辑，用户必须手动重新触发整个流程
- 设置页面没有重试次数配置项

### 约束条件

- 保持现有代码风格（中文命名混用）
- 不破坏现有存档兼容性
- 新增配置需要持久化到 IndexedDB

## Goals / Non-Goals

**Goals:**
1. 实现图片生成失败后保留提示词数据，供用户查看
2. 提供「完全重新生成」和「复用提示词重试」两个选项
3. 在设置界面添加可配置的默认重试次数
4. 在图片管理界面显示失败记录的提示词内容

**Non-Goals:**
- 不实现自动重试（用户手动选择重试方式）
- 不修改图片生成服务层的重试逻辑
- 不涉及移动端 UI 改动（本设计仅针对桌面端）

## 技术方案

### 1. 配置结构扩展

在 `utils/apiConfig.ts` 中的 `功能模型占位配置结构` 新增字段：

```typescript
// 扩展功能模型占位配置结构
interface 功能模型占位配置结构 {
  // ... 现有字段
  提示词生成重试次数?: number;      // 默认：1
  图片生成重试次数?: number;        // 默认：1
}
```

### 2. 数据模型扩展

在 `models/imageGeneration.ts` 中扩展生图结果类型：

```typescript
// 现有类型已有：生图词组、最终正向提示词、最终负向提示词
// 新增：是否保留提示词标志
interface NPC生图结果记录 {
  // ... 现有字段
  保留提示词?: boolean;  // 失败后是否保留提示词供重试
}
```

### 3. 工作流改造

#### 3.1 NPC生图工作流 (`npcImageWorkflow.ts`)

**修改点**：
- 在 catch 块中确保提示词数据已保存到最近生图结果
- 添加 `重试选项` 参数（`完全重试` | `复用提示词`）
- 添加 `重试次数` 参数，传递到图片生成函数

**流程变更**：
```
原流程：
  生成提示词 → 生成图片 → 成功/失败

新流程：
  生成提示词 → 生成图片
    ├── 成功 → 正常结束
    └── 失败 → 
        保留提示词 → 用户选择重试方式
          ├── 完全重试 → 重新执行整个流程
          └── 复用提示词 → 仅重新执行图片生成
```

#### 3.2 场景生图工作流 (`sceneImageWorkflow.ts`)

同上，修改模式一致。

#### 3.3 NPC秘档部位生图工作流 (`npcSecretImageWorkflow.ts`)

同上，修改模式一致。

### 4. UI 改动

#### 4.1 图片管理界面 (`ImageManagerModal.tsx`)

在失败记录的操作区增加：
- 「查看提示词」按钮 - 弹窗显示提示词内容
- 「完全重试」按钮 - 重新生成提示词和图片
- 「复用提示词重试」按钮 - 仅重新生成图片

提示词展示弹窗包含：
- 生图词组
- 最终正向提示词
- 最终负向提示词

#### 4.2 设置页面 (`ImageGenerationSettings.tsx`)

新增「重试设置」页签：
- 提示词生成重试次数：数字输入框（0-5，默认1）
- 图片生成重试次数：数字输入框（0-5，默认1）

### 5. useGame 层对接

在 `hooks/useGame.ts` 中添加新的 action：
- `handleRetryImage(npcId, options)` - 处理图片重试
- `handleRetrySceneImage(options)` - 处理场景图片重试

options 包含：
```typescript
interface 重试选项 {
  模式: '完全重试' | '复用提示词';
  重试次数?: number;
}
```

## 数据结构

### 新增配置字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 提示词生成重试次数 | number | 1 | 提示词生成失败时的重试次数 |
| 图片生成重试次数 | number | 1 | 图片生成失败时的重试次数 |

### 状态变更

**NPC生图结果记录**（失败时）：
- `生图词组`: string - AI 生成的提示词
- `最终正向提示词`: string - 完整正向提示词
- `最终负向提示词`: string - 完整负向提示词
- `状态`: 'failed'
- `错误信息`: string

## 影响评估

### 功能影响
- 扩展了图片生成失败后的用户交互
- 新增设置项需要用户配置

### 兼容性影响
- 配置字段有默认值，旧的存档无需迁移
- 现有工作流行为保持向后兼容

### 性能影响
- 提示词保存到现有状态结构中，无额外存储压力
- UI 改动对渲染性能无明显影响
