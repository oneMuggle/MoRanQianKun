## Context

references/气运.md 包含300+气运设定，需与项目境界系统适配。当前项目境界层级范围为0-33+（凡人→元婴），气运需标注适用区间。

## Goals / Non-Goals

**Goals:**
- 气运按能力类型分类（6类）
- 气运标注适用境界范围
- NSFW内容明确标记可过滤
- 效果分为可量化/描述性两类

**Non-Goals:**
- 不修改气运效果原设定
- 不做平衡性调整

## Technical Decisions

### 1. 数据结构扩展
在气运结构中增加字段：

```typescript
interface 气运结构 {
    名称: string;
    类别: string;           // 原类别（真·气运/限制版气运等）
    能力类型: '战斗' | '生存' | '社交' | '谋略' | '特殊' | '辅助';
    适用境界: [number, number]; // [最低境界层级, 最高境界层级]
    效果类型: '可量化' | '描述性';
    属性修正?: {...};       // 可量化时使用
    nsfw?: boolean;       // 明确标记
    代价?: string;       // 限制版气运的负面描述
}
```

### 2. 分类规则
根据效果描述判断能力类型：
- 直接提升战斗能力 → 战斗
- 保命/恢复/防御 → 生存
- 人脉/声望/威吓 → 社交
- 情报/判断/布局 → 谋略
- 奇遇/运气/抽奖 → 特殊
- 修炼/效率/资源 → 辅助

### 3. NSFW 标记规则
以下类别的气运标记为 NSFW：
- 涉及性行为、性暗示的内容
- 极度暴力、血腥描述
- 伦理禁忌内容（如亲属乱伦等）

### 4. 效果量化
可量化气运标注：
- 属性修正值（如力量+20%）
- 触发概率（如15%概率触发）
- 数值范围（如1-3个随机气运）

描述性气运保留原描述：
- 剧情效果、特殊机制
- 难以数值化的长期收益

## Implementation Plan

### Phase 1: 数据处理
- 解析 references/气运.md 原始数据
- 为每个气运标注能力类型
- 为每个气运标注适用境界范围
- 标记 NSFW 内容

### Phase 2: 效果分类
- 区分可量化/描述性效果
- 为可量化气运标注数值
- 保留描述性气运原描述

### Phase 3: 验证
- 按分类统计数量
- 验证 NSFW 标记准确性
- 验证境界适配合理性

## Impact Assessment

### 功能影响
- 气运数据结构字段增加
- 选择器需支持多维度筛选
- 现有气运数据需迁移

### 兼容性
- 旧存档无新增字段时使用默认值
- 新字段为可选

### 性能
- 数据量增加约20%（新增字段）
- 筛选时需遍历新增字段
