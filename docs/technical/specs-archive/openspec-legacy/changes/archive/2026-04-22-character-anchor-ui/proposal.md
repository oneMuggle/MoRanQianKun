## Why

当前角色锚点后端逻辑已完成（数据结构、CRUD、提取、装配），但缺少管理界面。用户无法可视化地管理已保存的角色锚点、编辑提示词、启用/禁用自动注入，也无法从 NPC 资料一键提取锚点。完成管理 UI 是角色锚点功能的最后一块拼图，直接决定功能能否真正服务于玩家。

## What Changes

- 新增**角色锚点管理面板**（桌面端 + 移动端），支持锚点的增删改查、启用控制
- 在图片管理/社交界面新增**提取角色锚点**入口，从 NPC 文本自动提取
- 支持手动编辑角色锚点的正负面提示词和结构化标签
- 支持重新提取锚点、手动编辑、删除等全生命周期操作
- 支持设为默认生图附加、场景生图自动注入的开关控制

## Capabilities

### New Capabilities

- `anchor-list-view`: 锚点列表视图 — 以卡片/列表形式展示所有角色锚点，显示关联 NPC、名称、启用状态、生成时间
- `anchor-detail-edit`: 锚点详情编辑 — 查看/编辑锚点的正面提示词、负面提示词、结构化标签，启用/禁用各注入选项
- `anchor-extract-action`: 锚点提取操作 — 从 NPC 资料一键提取或重新提取角色锚点
- `anchor-management-entry`: 锚点管理入口 — 在图片管理或社交界面提供明确的锚点管理入口

### Modified Capabilities

- （无）

## Scope

### 包含

- 桌面端角色锚点管理界面（ImageManagerModal 或独立面板）
- 移动端角色锚点管理界面
- 锚点列表展示与分页/筛选
- 锚点详情查看与编辑
- NPC 文本一键提取锚点
- 锚点启用/禁用开关（默认附加、场景自动注入）
- 锚点删除与确认

### 不包含

- 角色锚点的后端存储逻辑修改（已完成）
- 角色锚点的提取提示词与装配逻辑修改（已完成）
- 图片生成流程的修改（已完成）
- 词组转化器预设管理 UI（已有独立界面）
