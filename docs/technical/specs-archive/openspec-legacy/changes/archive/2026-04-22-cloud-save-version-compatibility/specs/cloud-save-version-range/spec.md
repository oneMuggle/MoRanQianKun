## ADDED Requirements

### Requirement: 云存档版本兼容性范围验证
云同步下载时，系统 SHALL 允许兼容版本范围内的云存档正常恢复，超出范围时给出明确错误提示。

#### Scenario: 版本在兼容范围内
- **WHEN** 用户从云端下载版本号为 1 的云存档，且当前代码兼容版本范围为 1~2
- **THEN** 存档正常恢复，不抛出错误

#### Scenario: 版本低于最小兼容版本
- **WHEN** 用户从云端下载版本号为 0 的云存档，且当前代码兼容版本范围为 1~2
- **THEN** 系统抛出错误，提示"云存档版本 0 不在兼容范围内 (1~2)，请重新上传一次云存档。"

#### Scenario: 版本高于最大兼容版本
- **WHEN** 用户从云端下载版本号为 3 的云存档，且当前代码兼容版本范围为 1~2
- **THEN** 系统抛出错误，提示"云存档版本 3 不在兼容范围内 (1~2)，请重新上传一次云存档。"

#### Scenario: 格式标识不匹配
- **WHEN** 用户尝试导入一个格式标识不是 `wuxia-cloud-sync-zip` 的 ZIP 文件
- **THEN** 系统抛出错误，提示"云存档格式不受支持"

### Requirement: 版本迁移日志
当云存档版本与当前版本不一致时，系统 SHALL 在控制台输出迁移日志。

#### Scenario: 版本需要迁移
- **WHEN** 存档版本与当前代码版本不一致但仍在兼容范围内
- **THEN** 控制台输出 `[云同步] 从版本 X 迁移到版本 Y...`
