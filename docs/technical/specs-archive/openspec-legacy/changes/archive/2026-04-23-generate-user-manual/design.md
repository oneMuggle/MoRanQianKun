## Context

项目已有 README.md（开发导向）、AGENTS.md（开发者知识库）、CONTRIBUTING.md（贡献指南），但缺少面向最终用户的操作手册。现有文档对普通用户不够友好：
- README.md 侧重技术栈和开发命令
- AGENTS.md 包含代码级细节，不适合普通用户
- 应用内功能面板 22 个模块 + 44 个工作流，缺乏使用指引

用户群体：主要是有一定游戏经验的玩家，希望快速上手武侠互动叙事游戏。

## Goals / Non-Goals

**Goals:**
- 生成结构清晰、易于理解的详尽用户手册
- 覆盖全部核心功能的入口位置和基本使用方法
- 提供常见游戏场景的操作指南
- 手册输出为静态 Markdown 文档，放置在 `docs/user-manual.md`

**Non-Goals:**
- 不创建运行时帮助系统（Tooltip、Hover Card 等）
- 不重构代码或修改现有功能
- 不涉及后端部署细节（已有 DEPLOY.md）
- 不翻译现有英文文档

## 技术方案

### 手册结构设计

参考 README.md 的目录结构和 PROJECT_STRUCTURE.md 的模块分类，用户手册按以下类别组织：

```
1. 快速开始
   - 游戏启动、界面布局、基本操作

2. 核心玩法
   - 开局创建、剧情推进、世界演变、战斗系统

3. 功能面板指南（按界面Tab分类）
   - 角色：属性、境界、功法
   - 物品：装备、背包
   - 社交：好友、NPC、互动
   - 门派：创建、升级
   - 任务：主线、支线
   - 世界书：编辑、使用

4. 图像系统
   - 角色生成、场景生图、图片管理

5. 存档与同步
   - 本地存档、云同步配置

6. 设置与配置
   - 模型选择、提示词调整

7. 常见问题
```

### 数据来源

- `components/features/AGENTS.md` - 22 个功能模块清单
- `hooks/useGame/AGENTS.md` - 44 个工作流概述
- `services/AGENTS.md` - 服务层能力（AI、同步）

### 输出路径

- `docs/user-manual.md` - 主手册文件
- 更新 `README.md` 添加手册链接

## 影响评估

| 维度 | 影响 |
|------|------|
| 功能 | 无影响，仅文档 |
| 兼容性 | 无影响 |
| 性能 | 无影响 |
| 代码 | 仅 README.md 链接更新 |

## Risks / Trade-offs

- [风险] 功能模块细节可能随版本变化 → 手册定位为静态文档，版本更新时需手动同步
- [风险] 用户手册篇幅可能过长 → 按主题章节拆分，TOC 索引便于跳转
