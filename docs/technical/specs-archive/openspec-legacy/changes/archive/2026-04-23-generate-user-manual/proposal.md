## Why

项目目前缺少一份面向最终用户的详尽使用手册。现有 README.md 仅包含开发和部署信息，而应用内功能面板复杂（22个模块 + 44个工作流），新用户难以快速理解核心玩法和各项功能的使用方法。迫切需要整理项目各部分的功能要点，生成用户手册帮助用户快速上手。

## What Changes

- 创建完整的用户手册文档（user-manual.md），涵盖所有核心功能的使用指南
- 在 README.md 中添加手册入口链接
- 按功能模块组织手册结构，便于用户快速定位

## Capabilities

### New Capabilities

- `user-manual`: 详尽用户手册，整合项目功能要点、使用指南和常见问题解答

### Modified Capabilities

（无）

## Impact

- 新增文件：`docs/user-manual.md` 或项目根目录用户手册
- 修改文件：`README.md`（添加手册链接）

## 变更范围

### 手册应覆盖的内容

- 核心游戏流程：开局创建、剧情推进、世界演变
- 功能面板：角色、装备、背包、功法、战斗、任务、社交、队伍、门派、世界书等
- 图像系统：角色生成、场景生图、图片管理
- 存档与同步：本地存档、云同步配置
- 设置与配置：模型选择、提示词调整、世界书编辑
- 开发者信息：技术栈、目录结构、贡献指南

### 非目标

- 不包含代码级开发文档（已有 AGENTS.md 和 CONTRIBUTING.md）
- 不创建运行时帮助系统（应用内 Tooltip 等）
- 不翻译除手册本身外的现有文档

## 验收标准

- [ ] 手册覆盖全部 22 个功能模块的基本使用说明
- [ ] 包含至少 5 个常见场景的操作指南
- [ ] 提供功能入口在哪里、核心概念是什么、如何使用的基本答案
- [ ] 手册语言为简体中文，结构清晰，分类合理
- [ ] README.md 中包含手册的访问链接
