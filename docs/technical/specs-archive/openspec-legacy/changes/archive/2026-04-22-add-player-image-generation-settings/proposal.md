# 主角生图设置优化提案

## Why

当前文生图设置面板中，NPC生图和场景生图都有独立的配置选项和设置页签，但**主角生图**完全缺失独立配置。用户无法：
1. 为主角选择独立的生图后端/模型
2. 配置主角专属的词组转化器/画师串预设
3. 管理主角角色锚点

这导致主角生图强制复用NPC生图设置，无法针对主角特点进行优化（如更精致的立绘需求）。

## What Changes

1. **新增「主角」设置页签** - 在文生图设置中新增与「NPC」「场景」并列的主角专属配置页签
2. **主角生图独立后端/模型** - 支持独立选择后端类型和模型
3. **主角词组转化器预设** - 支持选择主角专属的词组转化器预设
4. **主角画师串预设** - 支持选择主角专属的画师串预设
5. **主角PNG画风预设** - 支持选择主角专属的PNG画风预设
6. **主角角色锚点管理UI** - 在设置面板中提供主角锚点的查看/提取入口

## Capabilities

### New Capabilities
- `player-image-settings`: 主角生图专属设置页签及配置
- `player-anchor-management`: 主角角色锚点管理入口

### Modified Capabilities
- `image-generation-settings`: 扩展现有设置面板，添加主角页签

## Impact

- **修改文件**: `components/features/Settings/ImageGenerationSettings.tsx`
- **新增类型**: 可能需要在 `models/system.ts` 添加主角生图相关字段
- **配置持久化**: 设置结构变更需兼容旧数据

## 非目标

- 不修改游戏内触发主角生图的业务流程（仅修改设置配置）
- 不修改现有的 NPC/场景生图逻辑
- 不添加主角自动生图触发规则（本提案仅覆盖设置面板）

## 验收标准

1. 设置面板中存在「主角」页签，与NPC/场景页签并列
2. 主角页签可配置：后端类型、模型、词组转化器、画师串、PNG画风
3. 主角配置可保存并在下一次生图时生效
4. 主角角色锚点可从设置面板触发提取
5. 配置变更后主角生图使用新的配置（而非复用NPC配置）
