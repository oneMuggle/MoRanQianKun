## Why

当前项目世界观硬编码为古代中国(古代武侠背景)，无法支持其他时代背景的叙事体验。用户希望能够选择不同的时间背景(古代/现代/未来)，扩展游戏世界观的可能性。

## What Changes

1. **新增时代选择配置项** - 在游戏设置中添加时代选择器，支持：古代(默认)、现代、未来三种时代背景
2. **新增时代特定提示词模块** - 创建古代/现代/未来各自的底层逻辑提示词模块
3. **修改世界生成系统** - 世界生成时根据选择的时代动态加载对应提示词
4. **创建时代特定预设数据** - 各时代对应的背景、天赋、节日等预设数据
5. **UI支持** - 新建游戏时的时代选择界面

## Capabilities

### New Capabilities

- `era-selection`: 时代选择器配置 (古代/现代/未来)
- `era-specific-prompts`: 时代特定提示词模块 (古代/现代/未来各自的世界运行逻辑)
- `era-presets`: 各时代的背景、天赋、节日等预设数据
- `world-generation-era`: 时代感知的世界生成系统
- `era-ui`: 新建游戏时的时代选择界面

### Modified Capabilities

- `game-settings`: 扩展游戏设置结构，添加时代选择项
- `opening-workflow`: 开局工作流需要感知时代配置

## Impact

- **models/system.ts**: 扩展 `游戏设置结构`，添加 `时代` 字段
- **prompts/core/ancientRealism.ts**: 重命名为通用模块或添加时代变体
- **prompts/runtime/worldGeneration.ts**: 时代感知的提示词生成
- **data/**: 各时代对应的预设数据文件
- **components/features/**: 新建游戏时的时代选择UI
- **hooks/useGame/**: 开局和世界生成工作流需要感知时代

## 非目标

- 本次不实现古代/现代/未来完全不同的游戏机制系统
- 不修改核心战斗、社交、修炼等游戏机制
- 不添加过于科幻的未来武器/科技描写

## 验收标准

1. 用户在新建游戏时可选择时代(古代/现代/未来)
2. 选择不同时代后，世界生成使用对应的时代背景提示词
3. 各时代的底层逻辑(礼制/社会/科技等)符合所选时代特征
4. 古代背景可使用现有武侠世界观，现代/未来需要新的世界观设定
5. 现有存档不受影响，新存档可选择时代
