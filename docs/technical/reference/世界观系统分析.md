# 世界观系统分析文档

## 概述

本项目中的"世界观"系统包含两个层面：
1. **AI 生成的世界观文本 (world_prompt)** - 注入给 AI 的提示词内容
2. **游戏运行时世界状态 (世界数据结构)** - 随游戏推进动态更新的结构化数据

---

## 一、数据类型定义

### 1.1 核心世界结构 (`models/world.ts`)

```typescript
世界数据结构 {
  活跃NPC列表: 活跃NPC结构[]      // 后台追踪的不在场 NPC
  待执行事件: 世界待执行事件结构[] // 尚未开始但已具备条件
  进行中事件: 世界进行中事件结构[] // 已开始推进的事件
  已结算事件: 世界已结算事件结构[] // 结果已落定的事件
  世界镜头规划: 世界镜头结构[]    // 镜头外正在发生的事
  江湖史册: 世界史册条目结构[]    // 长期历史归档
  地图: 地图结构[]               // 地点坐标与归属
  建筑: 建筑结构[]               // 地点内部构造
}
```

**各结构详情：**

| 结构 | 核心字段 | 用途 |
|------|---------|------|
| 活跃NPC | 姓名/所属势力/当前位置/当前行动/行动时间 | 后台 NPC 动态追踪 |
| 待执行事件 | 事件名/计划执行时间/前置条件/触发条件/阻断条件 | 未来事件规划 |
| 进行中事件 | 事件名/开始时间/预计结束时间/当前进展/已产生影响 | 进行中事件追踪 |
| 已结算事件 | 事件名/结算时间/事件结果/长期影响/是否进入史册 | 结果沉淀层 |
| 世界镜头 | 镜头标题/触发时间/触发条件/沉淀内容 | 氛围与张力池 |
| 江湖史册 | 标题/归档内容/长期影响/关联人物 | 重大历史归档 |

### 1.2 世界书系统 (`models/worldbook.ts`)

```typescript
世界书条目 {
  id: string
  标题: string
  内容: string
  类型: 世界书类型        // 'world_lore' | 'system_rule' | 'command_rule' | 'output_rule'
  作用域: 世界书作用域[] // 'main' | 'opening' | 'world_evolution' | 'variable_calibration' ...
  注入模式: 'always' | 'match_any'
  条目形态?: 'normal' | 'timeline_outline' | 'time_injection'
  内置分类?: '常驻' | '开局' | '主剧情' | '变量生成' | '文章优化' | '回忆' | '世界演变'
  时间线范围?: [开始时间, 结束时间]
  优先级?: number
}

世界书 {
  id: string
  标题: string
  描述?: string
  条目: 世界书条目[]
  启用?: boolean
}
```

**世界书作用域对应链路：**

| 作用域 | 生效场景 |
|-------|---------|
| `main` | 主剧情生成 |
| `opening` | 开局生成 |
| `world_evolution` | 世界演变 |
| `variable_calibration` | 变量校准 |
| `story_plan` | 剧情规划 |
| `tavern` | 酒馆预设 |

## 二、世界观生成机制

### 2.1 生成链路 (`prompts/runtime/worldGeneration.ts`)

```
世界生成流程：
1. 构建世界观种子提示词 (worldSetup.ts)
2. 注入主角建档锚点信息
3. 调用 AI 生成 <世界观>...</世界观> 标签内容
4. 将生成结果写入提示词池 core_world
```

**世界生成提示词核心约束 (`世界观生成系统提示词`)：**

- **输出格式**：必须为 `<世界观>...</世界观>` 标签块
- **内容范围**：只允许"世界观信息"，不含 tavern_commands、命令格式、UI 说明
- **字数限制**：2500 字内软限制
- **必含信息**：
  1. 世界总览（名称/时代/地理/社会秩序/文明结构）
  2. 势力版图（立场/目标/冲突关系）
  3. 社会环境（治安/经济/江湖风气）
  4. 风险生态（冲突源头/危险区域）
  5. 成长体系边界（武侠口径，非仙侠）
  6. 世界运行逻辑

**关键约束**：
- 保持"长期母本"定位，非一次性开场剧情
- 世界地图采用绝对坐标体系 `1坐标单位=100米`
- 玩家初始点固定为坐标原点 `[0,0]`
- 不写入玩家姓名、背景、初始关系网
- 高手、秘籍、名器保持稀缺度

### 2.2 工作流实现 (`hooks/useGame/worldGenerationWorkflow.ts`)

```typescript
执行世界生成工作流(params) {
  // 1. 校验 API 配置
  // 2. 构建种子提示词
  // 3. 生成/校验境界体系提示词 (可选)
  // 4. 生成世界观提示词
  // 5. 写入提示词池 (core_world + 境界体系)
  // 6. 持久化到 IndexedDB
  // 7. 进入开局剧情生成 (mode='all') 或等待玩家指令 (mode='step')
}
```

## 三、世界演变机制

### 3.1 演变链路 (`prompts/runtime/worldEvolution.ts`)

世界演变是**每回合后台审计**，维护后台世界状态而不参与主剧情正文生成。

```
每回合演变流程：
1. 审计"是否需要更新"
   - 时间门槛触发
   - 行为导致世界层变化
   - 后台状态到期/断档
   - 小说分解滑窗逼近

2. 处理对象迁移
   待执行 → 进行中 → 已结算 → 江湖史册

3. 维护条数约束
   - 活跃NPC: 常态7条, 峰值9条
   - 进行中事件: 常态5条, 峰值7条
   - 已结算事件: 常态4条, 峰值6条

4. 刷新活跃 NPC 与世界镜头

5. 地图/建筑/环境联动
```

### 3.2 演变上下文构建 (`hooks/useGame/worldEvolutionUtils.ts`)

```typescript
构建世界演变上下文文本(params) {
  return `
  【世界观提示词】
  【世界演化规则】
  【当前游戏内时间】
  【当前环境】
  【当前世界】
  【当前剧情锚点】
  【本回合前台已发生事实】
  【本回合<剧情规划>】
  【本回合主链已落地命令】
  【短期记忆】
  【最近数回合前台回顾】
  【动态世界线索】
  【到期触发摘要】
  【本回合可触发演变候选】
  `
}
```

### 3.3 命令规范 (`prompts/runtime/worldDataSchema.ts`)

| 命令 | 用途 |
|-----|------|
| `set 世界.活跃NPC列表[i].当前行动` | 刷新已有项 |
| `push 世界.活跃NPC列表 = {...}` | 新增完整项 |
| `delete 世界.活跃NPC列表[i]` | 删除项 |
| `set 环境.时间 = "YYYY:MM:DD:HH:MM"` | 联动时间 |

**命令顺序**：`先 set/add → 再 push → 最后 delete`

## 四、提示词注入机制

### 4.1 核心提示词 (`prompts/core/`)

| 文件 | 用途 |
|-----|------|
| `world.ts` | 核心世界观占位符 (`核心_世界观`) |
| `realm.ts` | 境界体系提示词 |
| `rules.ts` | 核心游戏规则 |
| `format.ts` | 输出格式定义 |
| `cot.ts` | 主流程思维链 |

### 4.2 运行时提示词 (`prompts/runtime/`)

| 文件 | 用途 |
|-----|------|
| `worldGeneration.ts` | 世界生成系统提示词 |
| `worldEvolution.ts` | 世界演变系统提示词 |
| `worldEvolutionCot.ts` | 世界演变思维链 |
| `worldSetup.ts` | 世界种子/锚点构建 |
| `worldDataSchema.ts` | 世界数据结构参考 |
| `opening.ts` | 开局生成提示词 |
| `variableGeneration.ts` | 变量生成提示词 |

### 4.3 系统提示词构建 (`hooks/useGame/systemPromptBuilder.ts`)

系统提示词由多层提示词组装：
1. 核心规则 (`core/`)
2. 数值规则 (`stats/`)
3. 难度设定 (`difficulty/`)
4. 写作约束 (`writing/`)
5. **世界观提示词** (`core_world` + 世界书)
6. **境界体系提示词** (可选)

## 八、设计要点总结

1. **世界观文本 vs 世界状态分离**
   - `world_prompt` 是 AI 生成的世界描述，持久化在提示词池
   - `世界数据结构` 是运行时状态，随游戏推进动态更新

2. **长期母本定位**
   - 世界观设计为长期复用，非一次性开场
   - 支持坐标体系支撑地图探索与距离计算

3. **后台演变优先**
   - 世界演变不参与主剧情正文，只做后台维护
   - 通过条数约束防止状态膨胀

4. **结构化命令驱动**
   - 世界更新通过 `set/push/delete` 命令序列执行
   - 命令输出有严格顺序约束

5. **世界书灵活注入**
   - 支持作用域、注入模式、时间线多重控制
   - 可按场景动态加载不同世界书组合
