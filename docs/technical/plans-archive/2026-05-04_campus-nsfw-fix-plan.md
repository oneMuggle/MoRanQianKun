# 2026-05-04 校园NSFW深化系统完整修复方案

## 背景

校园NSFW深化系统的Prompt注入链路已打通（`时代配置ID` 传播链已修复），个性化欲望档案初始化已完成。但系统仍存在多个致命断裂点，导致欲望状态永远不会变化、存档后数据丢失等问题。

## 审计发现的问题

| # | 问题 | 严重程度 | 影响范围 |
|---|---|---|---|
| 1 | 存档保存时 `清洗导入存档` 剥离 `校园系统` 字段 | **致命** | 所有数据持久化 |
| 2 | 响应处理不支持 `校园系统` 根路径 | **致命** | AI状态更新回写 |
| 3 | 引擎函数38个全部为死代码 | **致命** | 所有NSFW子系统运转 |
| 4 | AI无输出状态更新的指令 | 严重 | 无法从AI获取状态更新 |
| 5 | 旧存档读档后欲望系统丢失 | 致命 | 存档兼容性 |
| 6 | `计算露出偏好推进` 未整合进 `处理NSFW互动` | 严重 | 露出子系统 |
| 7 | 多NPC只取焦点NPC注入prompt | 中等 | Prompt信息不完整 |
| 8 | `校园NSFW设置` 未纳入游戏设置默认值 | 低 | 新游戏配置 |

## 修复方案

### 修复 1：存档保存不剥离校园系统（致命）

**文件**：`services/dbService.ts`

**问题**：`清洗导入存档` 函数（约第142-184行）创建规范化存档对象时，没有包含 `校园系统`、`校规系统`、`催眠系统` 字段。

**修复**：在 `清洗导入存档` 的返回对象中添加这三个字段，使其在存档读写中保持完整。

### 修复 2：响应处理支持校园系统根路径（致命）

**文件**：`hooks/useGame/responseCommandProcessor.ts` + `utils/stateHelpers.ts`

**问题**：
- `responseCommandProcessor.ts` 的根路径列表中无 `校园系统`
- `stateHelpers.ts` 的 `applyStateCommand` 写回根路径 switch 中无 `校园系统`

**修复**：
1. 在 `responseCommandProcessor.ts` 的根路径列表中添加 `校园系统`
2. 在 `stateHelpers.ts` 的 `applyStateCommand` switch 中添加 `校园系统` 分支
3. 在 `stateHelpers.ts` 的 `extractStateCommands` 中添加 `校园系统` 的默认值

### 修复 3：AI Prompt 中增加状态输出指令（严重）

**文件**：`prompts/runtime/campusNSFW.ts`

**问题**：`构建校园NSFW完整叙事约束` 只告诉AI"当前是什么状态"，但没有要求AI在响应中返回更新后的状态。

**修复**：在约束末尾追加输出格式要求，指示AI在发生相关事件时输出 `<欲望系统状态>` 标签，包含更新后的 NPC 欲望档案数据（JSON格式）。

**输出格式示例**：
```xml
<欲望系统状态>
{
  "更新档案": {
    "NPC001": {
      "当前阶段": "试探",
      "阶段进度": 25,
      "关系轨道": "暧昧",
      "轨道进度": 15,
      "暴露风险值": 35,
      "流言等级": 1
    }
  }
}
</欲望系统状态>
```

### 修复 4：响应处理解析欲望系统状态更新（致命）

**文件**：`hooks/useGame/responseCommandProcessor.ts`

**问题**：没有解析 `<欲望系统状态>` 标签的逻辑。

**修复**：
1. 在 `responseCommandProcessor.ts` 的命令提取逻辑中添加对 `<欲望系统状态>` 标签的识别
2. 解析 JSON 内容，提取更新后的 NPC 欲望档案
3. 通过 `设置校园系统` 更新状态

### 修复 5：引擎函数在游戏循环中被调用（致命）

**方案选择**：本地计算 vs AI驱动

考虑到项目现有架构（AI通过响应命令更新游戏状态），采用 **混合模式**：

- **引擎函数作为计算工具**：在 `responseCommandProcessor.ts` 解析AI的状态更新后，调用引擎函数进行二次计算和校验
- **被动触发**：每回合自动衰减（流言、暴露风险）在回合结束时调用引擎函数

**具体修改**：

1. **在 `sendWorkflow/index.ts` 的响应处理阶段**：
   - 解析 `<欲望系统状态>` 后，调用 `处理NSFW互动` 的简化版本进行校验
   - 计算暴露风险自然衰减（每回合 -5）
   - 计算流言自然衰减（每回合 -2）

2. **整合 `计算露出偏好推进` 进 `处理NSFW互动`**：
   - 在 `campusNSFWEngine.ts` 的 `处理NSFW互动` 函数末尾调用 `计算露出偏好推进`
   - 将结果合并到返回的 `更新后档案` 中

### 修复 6：旧存档读档兼容（致命）

**文件**：`hooks/useGame.ts`

**问题**：旧存档读档后 `校园系统.欲望系统` 为空，初始化 useEffect 会重新创建但丢失所有历史数据。

**修复**：在读档成功后的回调中，检查 `校园系统.欲望系统` 是否存在，如不存在则基于当前社交列表中的主要角色重新初始化（保持与首次初始化相同的逻辑）。

### 修复 7：多NPC prompt注入增强（中等）

**文件**：`hooks/useGame/sendWorkflow/index.ts`

**问题**：`构建校园NSFW参数` 只取欲望阶段最高的NPC作为焦点。

**修复**：改为注入所有主要角色的欲望状态摘要，同时保留焦点NPC的完整数据。这样AI知道所有NPC的状态，但重点描写焦点NPC。

### 修复 8：校园NSFW设置纳入默认值（低）

**文件**：`utils/gameSettings.ts`

**修复**：在 `默认游戏设置` 中添加 `校园NSFW设置` 字段，引用 `models/campusNSFW/index.ts` 中已有的 `默认校园NSFW设置`。
