# NSFW 模块分析报告

## 1. NSFW 控制点总览

项目中存在 **四层相互独立但关联的 NSFW 控制系统**：

### 第一层：开局世界生成级 `nsfw场景类型`（4 档）
- **类型**: `'无' | '点到为止' | '适度展开' | '完全展开'`
- **存储位置**: `OpeningConfig.worldGenConfig.nsfw场景类型`（通过 NewGameWizard 设置）
- **作用**: 控制 AI 生成文本中 NSFW 场景描写的**文字风格档位**
- **影响范围**:
  - 天赋/背景的 `nsfw` 布尔过滤（`useNewGameWizardState.ts`）
  - 世界生成提示词中的 NSFW 场景描述注入（`worldSetup.ts`）
  - 双修门派世界书注入（`worldLixiangSects.ts`）
  - 运行时额外提示词组装（`nsfw.ts` → `构建运行时额外提示词`）
  - 文生图运行时提示词（`nsfw.ts` → `构建文生图运行时额外提示词`）
  - 亲密度动作约束（`intimacy.ts`）
  - NSFW 角色卡片（`nsfwCard.ts`）
  - 系统提示词组装中的 NSFW 卡片注入（`systemPromptBuilder.ts:1528`）

### 第二层：游戏运行时级 `启用NSFW模式`（开关）
- **类型**: `boolean`，默认 `false`
- **存储位置**: `游戏设置结构.启用NSFW模式`
- **作用**: 作为 **主开关** 控制是否注入 NSFW 提示词和显示 UI
- **影响范围**:
  - 系统提示词中 NSFW 角色卡片的注入（`systemPromptBuilder.ts:1528`）
  - 运行时额外提示词的 NSFW 部分激活（`nsfw.ts:82-86`）
  - 文生图运行时提示词的 NSFW 部分激活（`nsfw.ts:107-111`）
  - 女主私密档案 UI 显示（`App.tsx:1438,1449`）
  - 亲密提示词加载决策（`intimacy/index.ts:是否加载亲密提示词`）
  - 小说拆分运行时额外提示词（`novelDecompositionRuntime.ts:219`）
  - 变量模型工作流额外提示词（`variableModelWorkflow.ts:283`）
  - 开局故事工作流额外提示词（`openingStoryWorkflow.ts:617`）
  - 主剧情请求额外提示词（`mainStoryRequest.ts:152,154`）

### 第三层：气运系统级 `nsfw等级`（3 级）
- **类型**: `0 | 1 | 2`
- **存储位置**: `气运结构.nsfw等级`
- **配套开关**: `游戏设置结构.成人内容`（boolean，默认 `false`）
- **作用**: 控制气运（天赋特质）条目的可见性
- **影响范围**:
  - `applyQiyunFilters()` 在 `data/qiyun/index.ts:327-329`
    - `excludeNsfw: true` → 过滤掉所有 `nsfw等级` 有值的条目
    - `成人内容开启 === false` → 过滤掉 `nsfw等级 >= 2` 的条目
    - `nsfwOnly: true` → 只显示 `nsfw等级 > 0` 的条目
  - 新角色创建时随机气运（`useNewGameWizardState.ts:600`）：`excludeNsfw: true, 成人内容开启`

### 第四层（隐性）：天赋/背景级 `nsfw` 布尔
- **类型**: `boolean`（在 `天赋结构` 和 `背景结构` 中）
- **作用**: 标记单个天赋/背景是否为 NSFW 内容
- **影响范围**:
  - 开局向导中根据 `worldConfig.nsfw场景类型` 过滤选项（`useNewGameWizardState.ts:158,166`）

## 2. 控制点之间的逻辑关系

```
nsfw场景类型 (开局设置, 4 档)
    ├── 控制 AI 文本描写风格（无/点到为止/适度展开/完全展开）
    ├── 控制天赋/背景中 nsfw 标记条目的可见性
    ├── 控制双修门派世界书的注入强度
    ├── 控制亲密度动作约束的具体文案
    └── 作为参数传入构建运行时额外提示词()

启用NSFW模式 (游戏设置, 布尔开关)
    ├── 主开关：决定是否注入 NSFW 提示词
    ├── 决定是否构建 NSFW 角色卡片
    ├── 决定是否激活亲密提示词
    ├── 控制女主私密档案 UI 显示
    └── 所有运行时提示词构建都检查此开关

成人内容 (游戏设置, 布尔开关)
    └── 仅影响气运系统：决定是否显示 nsfw等级=2 的重口味气运
```

**关键依赖链**：
- `构建运行时额外提示词()` 需要 **同时** 接收 `启用NSFW模式` 和 `nsfw场景类型` 才能正确工作
- `nsfw场景类型` 来源于 `OpeningConfig`，**不在** `游戏设置结构` 中
- `启用NSFW模式` 来源于 `游戏设置结构`，**不在** `OpeningConfig` 中

## 3. 存在的问题分析

### 问题 1（HIGH）：`nsfw场景类型` 不在 `规范化游戏设置()` 中

`规范化游戏设置()` (`utils/gameSettings.ts`) 负责标准化所有游戏设置字段，但 **没有** 包含 `nsfw场景类型`。这意味着：

- `构建运行时额外提示词()` 在 `mainStoryRequest.ts:152` 接收的是 `runtimeGameConfig`（规范化后的 `游戏设置结构`）
- `游戏设置结构` 中 **没有** `nsfw场景类型` 字段
- 因此 `构建运行时额外提示词()` 拿到的 `options` 中 `nsfw场景类型` 是 `undefined`
- 回退逻辑 `options?.nsfw场景类型 || '完全展开'` 会 **永远使用 '完全展开'**

**影响**：只要 `启用NSFW模式 = true`，无论开局选择了什么档位（点到为止/适度展开），运行时 NSFW 提示词都会以 **完全展开** 的强度注入。开局设置的档位只对 UI 过滤和世界生成提示词有效，对运行时文本生成无效。

**同样的问题影响**：
- `构建文生图运行时额外提示词()` (`useGame.ts:1543`)
- `构建运行时额外提示词()` 在 `novelDecompositionRuntime.ts:219`
- `构建运行时额外提示词()` 在 `variableModelWorkflow.ts:283`

### 问题 2（HIGH）：`成人内容` 不在 `规范化游戏设置()` 中

`成人内容` 字段存在于 `游戏设置结构` 类型定义中，但 `规范化游戏设置()` 没有处理它。这意味着：
- 旧存档加载时，如果该字段不存在，不会正确回退到 `false`
- 规范化后的配置可能丢失此字段

### 问题 3（HIGH）：`启用里武侠模式` 不在 `规范化游戏设置()` 中

同样的问题，`启用里武侠模式` 字段没有被规范化处理。

### 问题 4（MEDIUM）：天赋/背景的 `nsfw` 布尔与气运的 `nsfw等级` 不统一

- 天赋/背景使用 `nsfw: boolean`
- 气运使用 `nsfw等级: 0|1|2`
- 两套系统不互通，导致过滤逻辑不一致：
  - 天赋/背景只看 `nsfw场景类型 !== '无'`
  - 气运看 `nsfw等级` 和 `成人内容开启`

### 问题 5（MEDIUM）：`nsfwCard.ts` 中 NSFW 角色卡片只过滤女性

```typescript
const 在场NPC = npcs.filter(npc => npc.是否在场 !== false && npc.性别 === '女');
```
仅女性 NPC 会被注入 NSFW 角色卡片，男性 NPC 被完全忽略。

### 问题 6（LOW）：`默认NSFW模式提示词` 与档位系统矛盾

`默认NSFW模式提示词` (`nsfw.ts:5-16`) 写死为 "extremely explicit" 和 "No euphemisms"，这与 "点到为止" 和 "适度展开" 档位的委婉要求矛盾。虽然 `构建里象修行叙事约束` 会覆盖部分行为，但基础提示词的强指令可能与档位约束冲突。

### 问题 7（LOW）：`intimacy/index.ts` 的 `是否加载亲密提示词` 未被调用

搜索结果显示 `是否加载亲密提示词` 函数在 `prompts/intimacy/index.ts` 中定义，但 **没有被任何其他文件引用或调用**。亲密提示词的注入可能通过其他路径（`nsfwCard.ts` + `intimacy.ts`）实现，导致此函数成为死代码。

## 4. 修复方案

### Phase 1: 修复规范化函数（核心修复）
- 在 `规范化游戏设置()` 中添加 `nsfw场景类型`、`成人内容`、`启用里武侠模式` 的规范化
- 在 `默认游戏设置` 中添加这些字段的默认值
- 确保 `规范化游戏设置()` 的返回类型包含这些字段

### Phase 2: 统一 NSFW 提示词矛盾
- 将 `默认NSFW模式提示词` 从 "extremely explicit" 改为中性描述
- 让具体档位通过 `构建里象修行叙事约束` 完全控制描写强度

### Phase 3: 清理死代码
- 移除或整合 `intimacy/index.ts` 中未被使用的函数
- 或将其接入到系统提示词组装流程中

### Phase 4: 统一天赋/背景/气运的 NSFW 标记
- 将天赋/背景的 `nsfw: boolean` 升级为 `nsfw等级: 0|1|2`
- 统一过滤逻辑

## 5. 风险评估

| 风险 | 级别 | 说明 |
|------|------|------|
| 运行时档位失效 | **HIGH** | 核心 bug，开局设置的档位无法在文本生成中生效 |
| 成人内容设置丢失 | **HIGH** | 旧存档加载时可能丢失成人内容设置 |
| 里武侠模式设置丢失 | **MEDIUM** | 旧存档加载时可能丢失里武侠设置 |
| 提示词指令冲突 | **LOW** | AI 可能同时收到矛盾指令，行为不确定 |
| 性别过滤偏差 | **LOW** | 仅女性有 NSFW 卡片，可能是有意设计 |
